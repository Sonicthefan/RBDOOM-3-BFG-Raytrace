REGIR-08: RLU-Owned Light Source Conversion
-------------------------------------------

Goal:

    Make the ReGIR module behave like r_pathTracingRemixLightUniverse: ReGIR
    owns only cell mapping and cell-local candidate slots, while the Remix
    Light Universe owns light identity, current/previous mapping, type ranges,
    bounded sample counts, and payload continuity.

    This task removes the active ReGIR route's ability to become a second light
    universe. It must not prove quality through temporal, PDFNEE, clean RTXDI,
    or debugMode 56. It is a current-frame source-domain repair only.

Reference file-equivalent:

    E:/prog/references/RTXDI-main/Libraries/Rtxdi/Include/Rtxdi/ReGIR/ReGIRSampling.hlsli
    E:/prog/references/RTXDI-main/Samples/FullSample/Shaders/LightingPasses/Presampling/PresampleReGIR.hlsl
    neo/renderer/NVRHI/PathTraceRemixLightManager.cpp
    neo/shaders/builtin/pathtracing/pathtrace_restir_local_debug_reservoirs.hlsli

Required rbdoom read set:

    neo/renderer/NVRHI/PathTraceRemixLightManager.cpp
    neo/renderer/NVRHI/PathTraceRemixLightManager.h
    neo/renderer/NVRHI/PathTraceSmokeSceneBuild.cpp
    neo/renderer/NVRHI/PathTraceSmokeDispatch.cpp
    neo/renderer/NVRHI/PathTraceReGIR.cpp
    neo/renderer/NVRHI/PathTraceReGIR.h
    neo/shaders/builtin/pathtracing/pathtrace_regir_debug.rt.hlsl
    neo/shaders/builtin/pathtracing/pathtrace_restir_local_debug_reservoirs.hlsli
    neo/shaders/builtin/pathtracing/RtxdiBridge/RAB_ReGIR.hlsli
    neo/shaders/builtin/pathtracing/RtxdiBridge/RAB_LightLoadRuntime.hlsli

Allowed rbdoom write set:

    neo/renderer/NVRHI/PathTraceSmokeDispatch.cpp
    neo/shaders/builtin/pathtracing/pathtrace_regir_debug.rt.hlsl
    neo/shaders/builtin/pathtracing/RtxdiBridge/RAB_ReGIR.hlsli
    docs/restir_regir_cleanroom

Required:

    ReGIR debug/build views 7, 9, and 10 require the current Remix/RAB light
        universe. If it is unavailable, the route must fail loudly or return an
        empty diagnostic result; it must not fall back to DoomAnalyticLights,
        SmokeEmissiveTriangles, the purged legacy light manager, or a local
        split-domain light table.
    candidate.lightIndex is the dense current RLU index
    candidate.globalIdentity is the dense current RLU index
    candidate.lightClass is derived from the RLU payload/type range
    analytic/emissive ranges come from the active RLU range helpers
    bounded per-type sample counts come from the active RLU sample-count
        helpers
    selected candidate replay uses the same active RRX RAB load/sample/PDF
        boundary as the functioning RLU route, not duplicated local light math
    r_pathTracingReGIRDump names:
        lightSource=current-remix-rab-light-universe
        whether legacy fallback is disabled
        current RLU count and type ranges
        selected dense current index
        selected type
        selected sourcePdf / invSourcePdf policy

Forbidden:

    no legacy split-array fallback in active ReGIR views 7, 9, or 10
    no deriving globalIdentity as emissiveCount + analyticIndex
    no local previous/current remap table owned by ReGIR
    no broad light suppression to hide unstable candidates
    no portal-only domain as the ReGIR source of truth
    no PDFNEE consume as proof
    no clean RTXDI temporal consume as proof
    no debugMode 56 / mode 72 as proof for this task
    no spatial reuse

Acceptance:

    With r_pathTracingReGIRLightDomain 2 and the RLU route active, ReGIR dumps
    show the same dense RLU source domain used by r_pathTracingRemixLightUniverse.
    Stationary debug views do not strobe because ReGIR is consulting a different
    light universe or local light identity scheme.
    If remaining flicker tracks animated payload radiance from RLU, the worker
    reports it as payload animation rather than identity/domain contamination.
    Stop after build/deploy for user visual testing.
