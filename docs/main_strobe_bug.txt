Yes. The bypass tells us the main failure is the persistent Doom analytic universe/remap contract, not RTXDI temporal itself.

Why accumulation works with bypass:

Current lights are uploaded as a compact active candidate table.
Previous lights are now retained in a small bypass-local previous table.
Current-to-previous remap is built by a stable key from the uploaded candidates.
RTXDI temporal can translate the selected current light to the correct previous light, so history is reusable.
Why the main one fails:

The persistent Doom analytic universe builds identities/remaps from a much larger authored/persistent light registry.
That registry includes broad static/dynamic state, animated/temporary/unproven entries, missing-current reasons, zero-radiance state, selected-area state, etc.
Its GPU remap says buffers are valid enough structurally, but something in that producer’s identity/remap/source-domain contract causes RTXDI temporal to reuse the wrong previous sample, reject valid history, or compare against an incompatible previous payload.
The visual proof is decisive: same clean RTXDI temporal/RAB/resolve works once that persistent universe layer is bypassed.
So the likely failing boundary is:

B/C: current reservoir valid, but persistent-universe identity/remap domain is wrong for temporal reuse.

Not:

final resolve, because ReferenceRab 9/10 did not fix the bad cases
raw Doom analytic shading, because bypass + color source works
RTXDI temporal generally, because the bypass accumulates
mirror/subview behavior
full-map brute force or spatial
The remaining main-path fix should audit BuildDoomAnalyticLightGpuRemap() and the persistent authored registry, especially:

candidate index assignment vs uploaded compact candidate order
currentCandidateIndex stored on persistent entries
previous entries using old currentCandidateIndex
remap validity conditions
dynamic/unproven/zero-radiance entries affecting remap table shape
selected-area/current-domain filtering mismatching previous-domain filtering
The bypass is not a compliant final solution, but it is a strong proof harness: preserve active compact sampling, preserve previous compact payloads, remap by stable key.
