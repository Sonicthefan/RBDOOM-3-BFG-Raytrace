B. EXISTING CLASSIFIER  (PathTraceDoomMaterialClassifier.cpp/.h, 389 lines)
--------------------------------------------------------------------------------
What it does today:
  - RtSmokeTranslucentClassifierInfo: a bag of bools driven by
      (1) MATERIAL-NAME token lists (guiTokens/glassTokens/glowTokens/
          decalTokens/signageTokens/particleTokens) — the reference doc's
          explicit DO-NOT list ("treat _d/_s/name as authoritative").
      (2) GL blend-state bits (additive / filter / src-alpha) per stage.
      (3) stage->lighting (SL_AMBIENT/SL_DIFFUSE) — partial role use.
  - Decision helpers: ResolveSmokeMaterialAlphaInfo (alphaTest+cutoff — this part
    IS metadata-correct, reads Coverage()/hasAlphaTest/alphaTestRegister),
    IsSmokeAdditiveDecalMaterial, IsSmokeAdditiveWhiteKeyMaterial,
    IsSmokeRgbKeyedBlendDecalMaterial, IsSmokeYCoCgDiffuseMapDecalMaterial.

Assessment:
  - GOOD and KEEP: the alpha-test/cutoff extraction and blend-bit inspection are
    legitimate metadata reads. Reuse the same logic in the new classifier, but do
    not move ownership into the legacy classifier or legacy facts record.
  - WEAK / TO REPLACE: surface-CATEGORY inference is 100% name tokens. This is
    where SURFTYPE_* must take over as the primary signal, with name tokens
    demoted to fallback-only (and flagged LOW confidence).
  - These helpers are pure (idMaterial -> info), so they are useful reference
    code. Keep the existing entry points stable for legacy callers; implement the
    new route/classification path in separate new module files.

D. LEGACY PERSISTENT FACTS  (PathTraceMaterialUniverse.h)
--------------------------------------------------------------------------------
RtSmokeMaterialUniverseFacts is the per-material persistent record (cached,
render-thread owned, keyed by materialId). It already stores: alphaTested,
alphaCutoff, diffuseYCoCg, additiveDecal/whiteKey, filterDecal, emissive,
emissiveColor/Luminance, has*Image/hasSafe*Texture booleans, fallbackAlbedo.

=> DO NOT add new classifier output here. This record is the legacy safety net.
   The new classifier should create its own persistent per-materialId record with
   equivalent cached lifetime/stability, and may copy useful legacy facts into
   that new record by value if needed.

--------------------------------------------------------------------------------
E. GPU MATERIAL STRUCT  (PathTraceEmissiveCandidates.h, PathTraceSmokeMaterial)
--------------------------------------------------------------------------------
Fields today: debugAlbedo[4], emissiveColor[4], {diffuse,alpha,normal,specular,
emissive}TextureIndex, alphaCutoff, flags (uint32), texture dims, padding0..2.
Flag bits live in PathTraceDynamicMaterialState.h (RT_SMOKE_MATERIAL_*).

MISSING for a proper classifier: roughness, metallic, IOR, transmission,
specularTint/F0. There are 3 padding words (padding0..2) — partial headroom, but
a clean BSDF block likely needs the struct widened. Any field add must be mirrored
in the HLSL material struct + keep 16-byte alignment, and triggers the shader-
rebuild gotcha (delete affected .rt.bin). See shader-rebuild-gotcha memory.

--------------------------------------------------------------------------------
F. TABLE BUILD  (PathTraceDynamicMaterialState.cpp, ~44k)
--------------------------------------------------------------------------------
BuildSmokeMaterialTableFromUniverse* turns universe facts -> PathTraceSmokeMaterial
rows + descriptor texture binding for the legacy path. The new path should have a
parallel row-fill function that writes the same shared GPU row type from the new
RtMaterialRecord. Cached by signature; the new-path signature must include the
new BSDF fields so edits/overrides invalidate correctly.
