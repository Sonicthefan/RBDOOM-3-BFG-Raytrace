ReSTIR PT Chain After Light Universe
Date: 2026-05-18

Purpose
-------
Define the intended RTXDI-shaped processing chain once rbdoom has a stable
unified light universe.

This document starts after Doom emissive triangles and Doom analytic lights
have been converted into one coherent light domain. It does not describe how to
build that light universe. It describes what should happen after it exists.


Starting assumption
-------------------
The light side has been fixed enough that rbdoom can provide:

    one current light count
    one previous light count
    one current light buffer
    one previous light buffer
    one current-to-previous remap
    one previous-to-current remap
    one local-light sampling mode
    one local-light source PDF story
    one RAB_LoadLightInfo(index, previousFrame)
    one RAB_SamplePolymorphicLight(...)

The first version can be minimal:

    emissive triangles + Doom analytic spheres only
    no environment light yet
    no infinite light yet
    uniform local-light sampling initially
    no local-light PDF texture initially

Do not keep pushing Doom-specific emissive/analytic branching deeper into every
RAB callback once this light universe exists. The Doom-specific work should be
before RAB, not inside every ReSTIR stage.


Chain overview
--------------
Target shape:

    rbdoom unified light universe
        -> RAB light callbacks
        -> RTXDI DI initial sampling for NEE
        -> optional secondary DI spatial resampling
        -> PT path tracer RecordNeeLightSample
        -> RTXDI PT initial reservoir
        -> RTXDI PT temporal resampling
        -> RTXDI PT spatial resampling
        -> RTXDI PT final shading input
        -> final shading / denoiser output

------------------------------------
Goal:
    Make rbdoom's unified light domain look like the reference application
    light domain.

    neo\shaders\builtin\pathtracing\RtxdiBridge\RAB_LightLoadRuntime.hlsli
    neo\shaders\builtin\pathtracing\RtxdiBridge\RAB_LightSamplingCore.hlsli
    neo\shaders\builtin\pathtracing\RtxdiBridge\RAB_LightSample.hlsli

RTXDI / FullSample reference files:
    E:\prog\references\RTXDI-main\Samples\FullSample\Shaders\LightingPasses\RtxdiApplicationBridge\RAB_LightInfo.hlsli
    E:\prog\references\RTXDI-main\Samples\FullSample\Shaders\LightingPasses\RtxdiApplicationBridge\RAB_LightSampling.hlsli
    E:\prog\references\RTXDI-main\Samples\FullSample\Shaders\LightingPasses\RtxdiApplicationBridge\RAB_Buffers.hlsli

    E:\prog\references\RTXDI-main\Samples\FullSample\Source\RenderPasses\PrepareLightsPass.cpp

Reference shape:
    RAB_LoadLightInfo(index, previousFrame)
        reads a coherent light buffer.

    RAB_SamplePolymorphicLight(lightInfo, surface, uv)
        samples a light record by type.

    RAB_EvaluateLocalLightSourcePdf(lightIndex)
        returns the local-light source PDF for that light index.

First rbdoom target:
    RAB_LoadLightInfo should become simple: index into unified current/previous
    light buffers, not branch across Doom domains.

    RAB_SamplePolymorphicLight should branch only on the unified light type.

    RAB_EvaluateLocalLightSourcePdf can start as uniform over the local-light
    region. Add a PDF texture later only if needed.

-----------------------------------
Goal:
    Replace rbdoom's hand-written local NEE RIS path with the same DI initial
    sampling primitive NVIDIA uses before recording PT NEE.

rbdoom files to replace or bypass:
    neo\shaders\builtin\pathtracing\RtxdiBridge\PathTracer\RAB_PathTracerNee.hlsli

RTXDI / FullSample reference files:
    E:\prog\references\RTXDI-main\Samples\FullSample\Shaders\LightingPasses\RtxdiApplicationBridge\PathTracer\RAB_PathTracer.hlsli
    E:\prog\references\RTXDI-main\Libraries\Rtxdi\Include\Rtxdi\DI\InitialSampling.hlsli
    E:\prog\references\RTXDI-main\Libraries\Rtxdi\Include\Rtxdi\DI\ReSTIRDIParameters.h
    E:\prog\references\RTXDI-main\Libraries\Rtxdi\Include\Rtxdi\RtxdiParameters.h

Reference functions:
    SampleDirectLightsForIndirectSurface(...)
    CalculateNEE(...)
    RTXDI_SampleLightsForSurface(...)
    RTXDI_SampleLocalLights(...)
    RTXDI_SampleInfiniteLights(...)
    RTXDI_SampleEnvironmentMap(...)
    RTXDI_SampleBrdf(...)

First rbdoom target:
    Construct minimal DI sampling inputs:

        RTXDI_DIInitialSamplingParameters
            numLocalLightSamples > 0
            numInfiniteLightSamples = 0
            numEnvironmentSamples = 0
            numBrdfSamples = 0
            localLightSamplingMode = Uniform
            enableInitialVisibility = true or current validated equivalent

        RTXDI_LightBufferParameters
            localLightBufferRegion.firstLightIndex = 0
            localLightBufferRegion.numLights = unified current light count
            infiniteLightBufferRegion.numLights = 0
            environmentLightParams.lightPresent = 0

    Then call RTXDI_SampleLightsForSurface for the current intersection
    surface and use the returned RTXDI_DIReservoir and RAB_LightSample.

Do not start with:
    environment sampling
    BRDF-hit light sampling
    local-light PDF texture
    ReGIR
    RIS presampling

Those can be added after the basic local-light DI path works.


Stage 3: Record selected DI result into PT
------------------------------------------
Goal:
    Feed the PT reservoir context using NVIDIA's CalculateNEE pattern instead
    of rbdoom's local RIS estimator.

rbdoom file:
    neo\shaders\builtin\pathtracing\RtxdiBridge\PathTracer\RAB_PathTracer.hlsli

RTXDI / FullSample reference files:
    E:\prog\references\RTXDI-main\Samples\FullSample\Shaders\LightingPasses\RtxdiApplicationBridge\PathTracer\RAB_PathTracer.hlsli
    E:\prog\references\RTXDI-main\Libraries\Rtxdi\Include\Rtxdi\PT\PathTracerContext.hlsli

Reference functions:
    CalculateNEE(...)
    ctx.RecordNeeLightSample(...)

Reference formula shape:
    if DI reservoir is valid:

        radianceFromLights =
            reflected BSDF radiance
            * visibility
            * RTXDI_GetDIReservoirInvPdf(diReservoir)
            / lightSample.solidAnglePdf

        sampledLightData.lightData = diReservoir.lightData
        sampledLightData.uvData = diReservoir.uvData
        neePdf = 1.0 / diReservoir.weightSum
        scatterPdf = RAB_SurfaceEvaluateBrdfPdf(...)
        ctx.RecordNeeLightSample(...)

First rbdoom target:
    Preserve the existing RAB_PathTrace loop shape, but make its NEE branch call
    a reference-shaped CalculateNEE equivalent instead of
    RAB_RecordSmokeNeeSample local RIS.

Do not invent alternate reservoir weights here.


Stage 4: Optional secondary DI spatial resampling

Reference function:

Reference inputs:

    current DI reservoir from RTXDI_SampleLightsForSurface

Immediate next correctness task after light universe
----------------------------------------------------
The first post-light-universe task should be:

    Replace RAB_RecordSmokeNeeSample's local RIS selection with a minimal
    reference-shaped CalculateNEE path that calls RTXDI_SampleLightsForSurface
    over rbdoom's unified local light region.

Strict constraints for that task:

    use uniform local-light sampling first
    local lights only
    no environment
    no infinite lights
    no BRDF-hit sampling
    no secondary DI spatial resampling
    no PSR
    no final resolve switch
    no custom reservoir math

Acceptance signal:
    Diagnostics show the PT initial reservoir NEE payload is populated by the
    RTXDI DI reservoir path, and the result is no worse than the old local-RIS
    path before enabling later features.
