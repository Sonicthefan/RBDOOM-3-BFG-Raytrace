ReSTIR PT RAB IO Supply Map
Date: 2026-05-17

Purpose
-------
Map each rbdoom RtxdiBridge file by input/output condition.

Category keys
-------------
Inputs:
    functional input
        Present input; supplier file exists; RAB can use it as intended.
    connected but wrong data
        Input is wired, but supplied data is unstable, incomplete, or not
        reference-valid.
    compromised input
        RAB callback was adapted to make do with rbdoom data instead of using
        the full NVIDIA reference input.
    missing input
        Required reference input has no real rbdoom supplier.

Outputs:
    clear function observed output correctly to ReSTIR
        Callback/output is connected and produces reference-shaped data for its
        supported domain.
    poisoned function thought to be broken from bad supplier data
        Callback shape is acceptable, but output can become invalid because
        supplied data is bad.
    compromised coding agent tried to bypass RBDoom limitation by inventing code and violating reference design
        Local adapter or substitute exists that is not NVIDIA-equivalent.
    missing output
        Code path exists, but output is stubbed, diagnostic-only, or inert
        because the supplier is absent.
