Task 03: Validated Temporal NEE Reuse
=====================================

Assignment
----------

Re-enable previous-frame NEE reservoir reuse only after stable light universe
upload and RAB remapping are implemented and validated.

Hard Acceptance Criteria
------------------------

Task 03 is accepted only if:

    Reuse-on output being quieter is not treated as proof.

    Diagnostics show correct remaps and bounded rejection reasons.

    The robot endpoint case does not corrupt unrelated current-frame lights.

    Rejection reasons distinguish:

        previous surface invalid
        previous reservoir invalid
        light remap invalid
        identity mismatch
        zero radiance
        invalid PDF/sample
        domain mismatch
        generation mismatch

    Dynamic/projectile/temporary lights without proven continuity reject
    previous-frame reuse.

Step-By-Step Procedure
----------------------

1. Remove temporary bypasses only when remap validation replaces them.

   Any bypass added to avoid unstable analytic NEE reuse should be replaced by:

       translate light index
       validate identity/generation
       validate light sampleability
       reject reservoir on failure

2. Keep the CVar default conservative until explicit acceptance.

   Task work starts from:

       r_pathTracingRestirPTTemporalAnalyticNeeReuse 0

   The default may move to 1 only after the user explicitly accepts the
   reuse-on behavior after testing.

3. Add a diagnostic mode or dump for rejection reasons.

   It should distinguish:

       previous surface invalid
       previous reservoir invalid
       light remap invalid
       light identity mismatch
       light currently zero-radiance
       light sample/PDF invalid
       domain mismatch
       generation mismatch

4. Validate with A/B runs.

   Compare:

       r_pathTracingRestirPTTemporalAnalyticNeeReuse 0
       r_pathTracingRestirPTTemporalAnalyticNeeReuse 1

   The reuse-on path must not corrupt current-frame lights when the robot stops
   or leaves the current view path.

5. Validate current-frame NEE still works.

   Modes 26, 27, 28, 32, 33, 50, and 51 should keep their existing debug
   meanings. Any output difference should be explainable through reservoir reuse
   rather than current light generation failure.

6. Update docs.

   If reuse-on is proven, update:

       docs/pathtracing.txt
       docs/pathtrace_modules.txt

   Document exactly which domains are remapped and which domains still reject
   previous reuse. After validation, the accepted default is
   `r_pathTracingRestirPTTemporalAnalyticNeeReuse 1`; keep the off state as a
   diagnostic split.

Validation
----------

Use a documented non-Debug PT lane.

Minimum tests:

    robot endpoint/stopped case
    robot on screen vs offscreen
    static area with stable Doom analytic lights
    strobed/animated light material
    emissive-only isolation with analytic candidates disabled
    temporal reuse off/on A/B

Expected result:

    reuse-on no longer makes unrelated lights flicker or turn off
    failed light remaps reject reservoirs
    current-frame NEE remains visible
    logs show bounded rejection reasons rather than slot reinterpretation

Stop Conditions
---------------

Stop and ask the user if:

    reuse-on still breaks current-frame lights after validated remap
    rejection is so aggressive that reuse contributes nothing
    emissive and analytic light domains collide
    the fix requires a full pass-graph rewrite
    the only apparent fix is to reset all history every animated-light frame

Non-Goals
---------

Do not make reuse-on the default without explicit user approval. Approval was
given after the robot/strobe flicker fix was validated, so the current branch
defaults to reuse-on.

Do not treat a visually quieter output as proof unless diagnostics show remaps
and rejects are correct.

Do not remove the off CVar; it remains the comparison and safety path.
