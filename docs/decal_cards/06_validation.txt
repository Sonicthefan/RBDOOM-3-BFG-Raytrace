VALIDATION / ACCEPTANCE GATE
============================
Authored: 2026-06-10. The gate DECAL-09 runs before flipping
r_pathTracingDecalComposite default 0->1. Every check is A/B against the cvar
(0 = today's stochastic path, must stay byte-identical; 1 = new composite).

SUGGESTED DIAGNOSTIC STAGING (from a prior attempt — isolates the three new pieces
so a failure points at one stage, not the whole path). Drive via an int cvar, e.g.
r_pathTracingDecalDebugStage:
  0 = off (stochastic path, baseline).
  1 = full composite ON (the production path).
  2 = NORMAL-OFFSET geometry only (DECAL-02): decals still shade via the old
      stochastic path, but the offset is applied — isolates z-fight/peeling from
      the composite. Use for V3.
  3 = ANY-HIT COLLECT only (DECAL-04): bin fills + base stays closest, but skip the
      composite (e.g. visualize bin count) — isolates accumulation from blending.
  4 = COMPOSITE diagnostic: tint/false-color the composited contribution so the
      blend op is visible independent of base texture — isolates the blend math.
Keep these as the debug ladder behind DECAL-08/08b; production ships stage 1.
