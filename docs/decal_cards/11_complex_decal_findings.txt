CASE 1 - textures/decals/alphabet4 (grey text renders as darkening)
-------------------------------------------------------------------
Material stage is `colored` + `blend diffuseMap` (SL_DIFFUSE). This is NOT a
filter/modulate decal: blend diffuseMap makes the stage a LIT DIFFUSE
INTERACTION layer. Raster result = litWall + light * tex.rgb * stageColor.rgb.
Black texels contribute zero (implicit mask); grey text picks up light color
and the `colored` per-entity tint (shaderParm0-3 / _color), which is why the
same texture reads white or yellow by placement.
OUR BUG: IsSmokeTranslucentOverlayCardMaterial tags it filterDecal (it is
sort=decal + ambientBlend-ish heuristics) -> composite MODULATES base albedo by
grey text -> darkening. Correct composite in our material-layer terms:
    albedo = saturate(albedo + tex.rgb * stageColor.rgb)   (additive ALBEDO,
    not emissive - the layer is lit, it is not a light source)
Tint source = evaluated stage color registers (drawSurf->shaderRegisters at
capture; parm-driven, per-instance, usually constant after spawn).

CASE 2 - textures/hell/pentastic1_spectrum (pulse-once emissive overlay)
------------------------------------------------------------------------
Dump: sort=2 (decal), polyOffset=1, 2 stages (bump _flat + DIFFUSE
pentastic1), filterDecal=1 blackKey=1 under current classifier, dump-time
stage color (1,1,1,1), isDynamic printed 0.
Same mis-route as case 1 (diffuse interaction stage treated as modulate), PLUS
animation: the pulse must come from evaluated stage color registers changing
per frame (and/or condition register turning the stage off = invisible state).
The dump's color=(1,1,1,1) is the materials own constant-register evaluation;
the live value needs drawSurf->shaderRegisters each frame.
=> Needs a PER-FRAME evaluated color/condition channel for detail-decal
materials, not a capture-time bake. Invisible state must DROP the layer
(condition==0 or color==black with diffuse-lit kind -> zero contribution
naturally).

CASE 3 - textures/decals/splat16 (post-trigger decal, invisible to new path)
----------------------------------------------------------------------------
Dump: 5 stages. stage0 AMBIENT splat16 (the real visible art), stage1
makealpha(splat16), stage2 reflect-cube2 env/genb, stage3 bump
heightmap(splat16,1), stage4 DIFFUSE 'guis/assets/white'.
KEY FINDING: texture discovery picked the stage-4 diffuse image
'guis/assets/white' and REJECTED it ("rejected dynamic image name" - gui-like
name filter) -> material has NO diffuse texture -> SampleSmokeDiffuseTexture
returns debugAlbedo. The 'solid white texture on top' the user sees in legacy
modes IS guis/assets/white - it is a lighting MASK for the diffuse interaction,
not art. The art is the ambient stage0 image.
The card is added by a trigger -> it is captured as DYNAMIC geometry (or
late-promoted static). Two gaps:
  (a) the face-normal offset only runs in the STATIC capture pass -> dynamic
      decal card z-fights (legacy modes show exactly that) and its any-hit
      collect is a coin flip -> invisible/unstable in composite mode.
  (b) even when collected, the composite samples the (missing) diffuse ->
      debug albedo, not splat16.
Also note: dynamic surfaces get VARIANT material ids
(SmokeRuntimeMaterialTableIdForDrawSurf) - the variant record must inherit
DETAIL_DECAL flags or the any-hit will not collect it. VERIFY.
'no material=' in the dump just means the condump missed the first line;
not evidence of an implicit material. The heightmap(...) normal is ordinary
idtech4 image-program syntax.

DYNAMIC CHANNEL INVESTIGATION (done)
------------------------------------
The engine ALREADY has the needed per-frame channel - do not rebuild it:
  - Capture: SmokeStageUsesPerSurfaceMaterialState (PathTraceSceneCapture.cpp:925)
    selects stages whose condition/color/alphaTest/texMatrix registers depend on
    runtime (parms or time). `colored` qualifies. Each such drawSurf gets a
    per-entity VARIANT material id (SmokeRuntimeMaterialTableIdForDrawSurf:1495)
    and an RtSmokeDynamicMaterialEvalSample with the EVALUATED color/condition
    from drawSurf->shaderRegisters.
  - Upload: BuildSmokeDynamicMaterialRecords (PathTraceSmokeSceneBuild.cpp:287)
    -> PathTraceDynamicMaterialRecord {color, texMatrix0 (.w=condition),
    texMatrix1 (.w=alphaTest), materialIndex, flags} -> dynamicMaterialBuffer.
    Count: m_sceneInputs.materials.dynamicMaterialRecordCount.
  - GPU: clean DI binds it as SmokeDynamicMaterials t15 (sentinel layout only).
    The PRODUCER's shared layout has Vulkan slot 15 taken by a UAV; free slot
    chosen for producer binding: StructuredBuffer_SRV(76) (74,75,77 used; 76
    free). Record count passed via new decalInfo2.x constant.
  - IMPORTANT dump insight: pentastic1_spectrum and splat16 dumps show
    class=particle/alpha -> these decals are captured in the DYNAMIC pass
    (instance 1) every frame, with VARIANT material indexes. That is why the
    static-pass-only offset never applied to them. The decal bin already stores
    the variant materialIndex, so a record lookup by decalMaterialIndex works.
  - Variant texture-info inheritance verified next (RegisterSmokeMaterialTextureVariant,
    PathTraceTextureRegistry.cpp:206).
