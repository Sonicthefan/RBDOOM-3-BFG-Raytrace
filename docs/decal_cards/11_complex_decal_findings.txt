CASE 1 - textures/decals/alphabet4 (grey text renders as darkening)
-------------------------------------------------------------------
Material stage is `colored` + `blend diffuseMap` (SL_DIFFUSE). This is NOT a
filter/modulate decal: blend diffuseMap makes the stage a LIT DIFFUSE
INTERACTION layer. Raster result = litWall + light * tex.rgb * stageColor.rgb.
Black texels contribute zero (implicit mask); grey text picks up light color
and the `colored` per-entity tint (shaderParm0-3 / _color), which is why the
same texture reads white or yellow by placement.
OUR BUG: IsSmokeTranslucentOverlayCardMaterial tags it filterDecal (it is
sort=decal + ambientBlend-ish heuristics) -> composite MODULATES base albedo by
grey text -> darkening. Correct composite in our material-layer terms:
    albedo = saturate(albedo + tex.rgb * stageColor.rgb)   (additive ALBEDO,
    not emissive - the layer is lit, it is not a light source)
Tint source = evaluated stage color registers (drawSurf->shaderRegisters at
capture; parm-driven, per-instance, usually constant after spawn).

CASE 2 - textures/hell/pentastic1_spectrum (pulse-once emissive overlay)

naturally).

SPECTRUM DISCOVERY (the pentastic pulse mechanism, confirmed from hell.mtr):
pentastic1_spectrum = DECAL_MACRO + `spectrum 1` + single `blend diffusemap`
stage with CONSTANT registers. The pulse is NOT in the material - a light
with `spectrum 1` animates, and spectrum surfaces are only illuminated by
matching-spectrum lights (idtech4 invisible writing).
  IMPLEMENTation: discovery stores info->detailDecalSpectrum =
  material->Spectrum(). BuildSmokeDynamicMaterialRecords (SceneBuild) now
  synthesizes a record per in-table spectrum detail decal: color = brightest
  matching-spectrum view light color (evaluated from the light shader
  registers x r_lightScale), STAGE_ENABLED only when such a light is lit.
  The record carries SELECTED_EMISSIVE; the composite routes such layers to
  emissiveRadiance (glows/pulses with the light, vanishes with it) instead of
  albedo.

SPLAT ROUTING FIX: when the ambient-stage diffuse rescue fires, the material
now routes OVER (filter flags cleared) with coverage from its makealpha()
stage image wired as the alpha texture - matching the raster src-alpha blend
of the art stage, instead of modulate/blackKey darkening.

FIELD-TEST ROUND 2 (emissive_escaped_animation/glow_instead_of_overlay/
alphabet_wrong_color dumps)
------------------------------------------------------------------------
Spectrum reveal worked but flickered green/blue, ran a phantom second cycle,
and sibling spectrum decals in the room ran the wrong/no animation.
ROOT CAUSE (not ID churn - the variant ids are per-entity stable; tableIndex
churn 196->120 across runs is same-frame-consistent): the spectrum scale was
a GLOBAL per-spectrum-number max over all in-view matching lights. The room
has several spectrum-1 lights with independent animations; every decal pulsed
with whichever light won the max that frame (flicker = differently-colored
lights trading the max; the phantom cycle = a second light bleeding in).
ALSO: spectrum decals with runtime registers (spectrum_symbol_letters_01,
isDynamic=1) get per-entity EVAL records, and the synthesis SKIPPED rows with
valid records -> those decals never received spectrum gating at all ("escaped
animation": always-on lit layer).
FIXES (round 2):
  - RtSmokeDynamicMaterialEvalSample now carries a representative world
    position (drawSurf tri bounds center -> world) per variant.
  - BuildSmokeSpectrumLights collects ALL matching-spectrum view lights with
    origin/radius/evaluated color (luminance capped at 8); a currently-dark
    light still claims its volume so its decal goes dark with it.
  - ChooseSmokeSpectrumLightColor picks the nearest light by volume-normalized
    distance when the surface position is known (per-entity variants), else
    the brightest match (constant-register base rows like pentastic).
  - Spectrum gating now also applies ON TOP of eval records (multiplied into
    record color, SELECTED_EMISSIVE set, disabled when the chosen light is
    dark) - no more skipped rows.

alphabet4 WRONG COLOR root cause: the eval record's generic stage selection
picks the highest-luminance stage; the white bump stage (1,1,1) beats the
authored yellow diffuse (0.949,0.561,0.200) seen in the dump. The sample now
carries the DIFFUSE stage's evaluated color/condition separately and
detail-decal records use it.
