Paste-Ready Worker Prompts
==========================

Use one prompt per worker. Replace CURRENT_BRANCH if needed.

Do not use old LM prompts for new work. The active sequence is LU-RM.

Every worker final response must include proof status:

    active consumer proof
        exact file/line evidence that the new output is read by the next active
        rbdoom module, shader binding, RAB callback, or dispatch path

    negative-path proof
        exact behavior when mapping/identity fails. For light/reservoir work,
        failed translation must reject only the selected candidate/sample

    runtime diagnostic proof
        dump/debug evidence separating storage survival, mapping failure,
        payload churn, global clear, and inactive consumer state

If a task is explicitly scaffold-only, the worker must say these proofs are not
yet present. Do not claim feature parity from class names, buffers, dumps, raw
storage, or a non-crashing build.
