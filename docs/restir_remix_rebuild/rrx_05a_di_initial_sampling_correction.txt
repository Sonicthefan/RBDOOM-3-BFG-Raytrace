RRX-05A DI Initial Sampling Corrective Slice
============================================

Purpose
-------

Correct the inactive rbdoom Remix-style DI initial sampling shader so it
matches the RTX Remix initial sampling pass shape more closely. This is a
correction slice for RRX-05, not a new feature slice.

The previous worker removed the empty placeholder, but the implementation still
has important contract failures:

    P1: the previous-best light can be sampled twice
    P1: explicit initial visibility is represented by a facing/geometry test,
        not real selected-light visibility
    P2: random-light sampling is still RTXDI SDK-style range delegation, not the
        RTX Remix per-type strided loop shape
    P2: portal index is carried in the contract but silently ignored

Do not use the existing rbdoom shader as proof that these choices are correct.
Use it only as the file to repair.

Required Correction Procedure
-----------------------------

1. Before editing, inspect the Remix initial pass and write down its exact pass
   shape in the worker response:

       load current surface
       use motion vector to find previous best-light source pixel
       translate previous best-light from previous-domain to current-domain
       build random-light reservoir
       build single previous-best-light reservoir
       combine best-light reservoir into random-light reservoir
       optionally update selected reservoir visibility
       force M = 1
       store to temporal output reservoir page

2. Inspect RTXDI helper functions before writing reservoir code. Use helper
   primitives such as:

       RTXDI_StreamSample
       RTXDI_CombineDIReservoirs
       RTXDI_FinalizeResampling
       RTXDI_StreamLocalLightAtUVIntoReservoir
       RTXDI_StoreVisibilityInDIReservoir

   Do not hand-roll RIS weights, target PDF math, MIS math, or finalization.

3. Replace the current random-light range delegation with a Remix-shaped loop:

       iterate the rbdoom light ranges by light type
       skip empty ranges
       create one local reservoir per range
       choose a random starting light inside that range
       advance by a deterministic stride based on range count and sample count
       wrap inside the range
       sample a light UV per candidate
       stream candidates through RTXDI helper primitives
       finalize the local reservoir with the local reservoir M
       set local reservoir M to 1
       combine the local reservoir into the pass reservoir

   The rbdoom ranges are:

       RemixRAB_GetEmissiveRange()
       RemixRAB_GetDoomAnalyticRange()

   The rbdoom sample counts are:

       RemixRAB_GetEmissiveSampleCount()
       RemixRAB_GetDoomAnalyticSampleCount()

4. Exclude the translated previous-best light from the random-light pass.

   If previous-best translation succeeds, the random-light loop must not also
   stream the same current light with the same portal domain. Remix does this so
   the previous-best light is represented by the single-light reservoir path,
   not double-counted by both paths.

5. Implement the single previous-best-light reservoir with RTXDI helper math.

   It must:

       reject invalid translated light indices
       sample the selected light once
       compute target PDF through the RAB light-sampling bridge
       stream/build the reservoir using RTXDI helper primitives
       set M = 1

   It must not be mixed into the random-light loop.

6. Fix initial visibility semantics.

   If real selected-light visibility tracing is not available in this inactive
   slice, do not claim explicit initial visibility is implemented. Either:

       add a clearly named deferred-visibility path that leaves visibility
       unchanged and documents that exact shadow visibility is deferred, or

       wire an existing rbdoom-compatible real visibility callback if one
       already exists in the allowed bridge layer.

   A facing/geometry check may be used only as candidate validity or target-PDF
   gating. It must not be presented as RTX Remix explicit selected-reservoir
   visibility.

7. Fix portal-index handling.

   Current rbdoom portal-domain support is not implemented in this inactive
   shader. Therefore:

       reject non-invalid previous-best portal indices with an explicit comment,
       or implement the matching surface/light transform path if the bridge
       already supports it.

   Do not silently carry a portal index and ignore it during sampling.

8. Keep the shader inactive unless the user separately commissions dispatch
   hookup. This slice corrects the shader contract only.

Acceptance Checklist
--------------------

The slice is acceptable only if:

    valid surfaces with valid light ranges no longer store only an empty
    reservoir

    previous-best translated light is excluded from random-light candidates

    previous-best single-light reservoir is separate and combined afterward

    random lights use per-range strided sampling rather than a flat or delegated
    all-range approximation

    visibility behavior is accurately represented as real visibility or
    explicitly deferred

    non-invalid portal indices are rejected or actually handled

    final pass reservoir is finalized through RTXDI helpers and M is forced to 1

    no old smoke sampling/reservoir headers or globals are introduced
