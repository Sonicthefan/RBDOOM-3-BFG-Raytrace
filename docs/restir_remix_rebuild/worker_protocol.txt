Worker Protocol
===============

Workers have repeatedly failed when asked to improvise ReSTIR behavior. Use
this protocol for every worker in this folder.


Mandatory First Step
--------------------

Before editing code, the worker must write a short local plan in its response:

    Remix file-equivalent being matched
    rbdoom file being edited
    exact input contract
    exact output contract
    forbidden areas it will not touch

If the worker cannot name those five items, it must stop.


Clean-Room Direct Port Rule
---------------------------

RTX Remix is the authoritative file responsibility graph. RTXDI is the
authoritative reservoir math contract. Do not copy source text. Do not redesign
algorithms.

Use "reference" only to identify source files that were inspected. It is not
permission to invent alternate behavior.

When translating a Remix file-equivalent:

    keep the same responsibility
    keep the same dataflow shape
    keep the same invalid/missing mapping behavior
    keep the same reservoir ownership idea
    adapt only the engine-specific data source

If the worker cannot identify the exact Remix file-equivalent, it must stop
before editing. Do not replace a Remix responsibility with an rbdoom smoke
reservoir equivalent.


Forbidden Patterns
------------------

Workers must not:

    tune around black/noisy output
    invent new ReSTIR math
    add broad dynamic-light suppression
    route new ReSTIR reservoirs through old smoke reservoir clear policy
    use Doom analytic count changes as temporal validity
    remove legacy outputs unless explicitly commissioned
    claim DLSS RR validation before DI/GI temporal debug views are sane


Required Review Output
----------------------

Every worker final response must include:

    files changed
    Remix file-equivalent matched
    exact contract added
    any remaining old-smoke dependency
    build result or "not built"
    runtime CVars to test, if applicable


Build And Deploy
----------------

Build from:

    E:/prog/rbdoom-3-bfg-rt/neo

Build command:

    cmake --build --preset win64-pt-dev-release

Deploy exe:

    copy E:/prog/rbdoom-3-bfg-rt/build-win64-pt-dev/Release/RBDoom3BFG.exe
    to E:/prog/rbdoom-3-BFG-prebuilt/RBDoom3BFG.exe

Deploy shaders after shader changes:

    copy the contents of E:/prog/rbdoom-3-bfg-rt/base/renderprogs2
    to E:/prog/rbdoom-3-BFG-prebuilt/base/renderprogs2

Do not create:

    E:/prog/rbdoom-3-BFG-prebuilt/base/renderprogs2/renderprogs2
