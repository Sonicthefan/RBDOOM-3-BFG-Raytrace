Obscure Issue Log
=================

Purpose
-------

This file records semi-obscure ReSTIR/RTXDI/Remix rebuild issues that are worth
preserving for future agents, but do not currently justify a dedicated fix
slice. These notes are guardrails. Do not treat them as permission to widen a
worker task.


RRX-05A: Conservative Visibility Callback Names
-----------------------------------------------

Status:

    Recorded only. No required fix while DI initial visibility remains deferred.

Files:

    neo/shaders/builtin/pathtracing/remix_rtxdi/rtxdi_initial_sampling.rt.hlsl

Issue:

    Rtxdi/DI/InitialSampling.hlsli expects RAB conservative visibility callback
    symbols to exist. The inactive rbdoom Remix DI initial shader currently
    keeps RAB_GetConservativeVisibility callbacks backed by a facing/geometry
    test.

Risk:

    Future agents may mistake these callbacks for RTX Remix explicit selected
    light visibility and route them into the initial-visibility path. That would
    silently reintroduce the earlier bug where a geometry-facing test is treated
    as real shadow visibility.

Current safe state:

    RRX-05A defers initial visibility through RemixRtxdiDeferInitialVisibility.
    The conservative callbacks are therefore callback-compatibility scaffolding,
    not the active visibility implementation.

Preferred future handling:

    Keep the callbacks only as RTXDI symbol compatibility until a real rbdoom
    visibility/shadow bridge exists. If a later slice enables explicit initial
    visibility, it must call a real selected-light visibility path equivalent to
    Remix RTXDI_UpdateSelectedReservoirVisibility behavior, not this geometry
    test.

Do not:

    remove the callbacks without checking RTXDI header compile requirements
    use the callbacks as proof that explicit visibility is implemented
    convert the facing/geometry test into reservoir visibility

view 68 quirks

    View 68 writes a clean RRX DI current-page seed from the active Remix/RAB light manager before temporal reuse. If no valid
    RRX DI initial seed can be produced, the view reports local rejection instead of fabricating input from old PT/spatial output.
    The view-68 current seed follows the Remix-shaped initial sampling pattern: frame-indexed RTXDI RNG, strided traversal of
    the active light-manager range, random local-light UVs, all candidates streamed, and one final RTXDI_FinalizeResampling call.
    When reprojection is not eligible, the active temporal path keeps the current RRX DI seed as current-only output instead
    of calling the temporal core with zero motion or clearing the reservoir.
    The left-half green status is previous-light translation valid for the selected previous candidate; it is not assigned from
    M/history growth. Right-half green still visualizes output M/history as a numeric heat channel.
    PathTraceRemixRtxdiResourceGate.* is only the resource/clear predicate owner. It is not a standalone Remix RTXDI
    temporal dispatch boundary.
    The RRX DI clear gate must include normal mode 56 when r_pathTracingRemixRtxdiResourcesEnable is set; otherwise the
    initial clearPending state keeps RestirPTRrxDiReservoirAvailable false and prevents the active consumer from starting.
    Normal mode 56 now consumes the RRX DI temporal reservoir as the direct-lighting contribution when that path produces a
    valid reservoir, and falls back to the old spatial direct-lighting path only when RRX DI is unavailable or invalid.
    The RRX DI consumer is still rbdoom's local preview/final resolve bridge, not full Remix DI parity. It applies rbdoom
    preview visibility via TraceSmokeShadowVisibility, not the RAB selected-light visibility callback or reference-equivalent
    final-shading contract.
