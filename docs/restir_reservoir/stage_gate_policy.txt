ReSTIR Reservoir Stage Gate Policy
Date: 2026-05-16

Purpose
-------
This policy prevents agents from racing through visual renderer work.

Hard Stop Rule
--------------
Every commissioned item must be one micro-slice. Build, deploy, report, and
stop. The next micro-slice requires a new user request after visual inspection.

If a task name contains "Task 01" but does not name 01A, 01B, 01C, 01D, or 01E,
do only 01A or ask the user to choose a micro-slice.

Visual Acceptance Rule
----------------------
An agent cannot mark a visual-quality slice as accepted. Only the user can.

Agents may report objective facts:

- build succeeded or failed,
- which pass ran,
- which CVar enables the debug view,
- whether a reservoir is valid/invalid by debug color,
- whether a new output is darker/brighter by obvious numeric preview.

Agents must not decide:

- this looks better,
- this is production-ready,
- this fixes DLSS/RR,
- this is visually acceptable.

Reference-First Rule
--------------------
Agents must not invent ReSTIR PT algorithms or substitute ad hoc reservoir math
for reference-defined processing. Temporal reuse, spatial reuse, neighbor
validation, target-function evaluation, normalization/bias correction, and
final shading must be mapped from the local RTXDI / RTX Remix reference
implementations into branch-local code.

When a reference step cannot be applied directly to rbdoom-3-bfg-rt, the agent
must identify the exact mismatch before implementing a local adaptation. The
adaptation must preserve the reference intent and be documented in the slice
summary or repo docs. Debug-only approximations are allowed only when explicitly
labeled as approximations and must not be promoted to production behavior.

Stage 01 Micro-Slices
---------------------
01A: Audit and map current GI flow.

    No code by default. Produces the route map and proposes 01B.

01B: Visual compare diagnostics only.

    Adds comparison views. Does not change normal mode 56 or RR input.

01C: Prove real GI temporal reuse only.

    Exposes temporal candidate in debug view. Does not add spatial or switch
    resolve.

01D: Prove GI spatial reuse only.

    Exposes spatial candidate in debug view. Does not switch resolve.

01E: Switch mode 56 resolve.

    Allowed only after user says 01D output is visually acceptable.

Later Stages
------------
Stage 02: RR-safe output shaping.

    Only after 01E is accepted.

Stage 03: RTX Remix-style hardening.

    Only after baseline GI reuse is visually accepted.

Stage 04: Unified DI/GI reservoirs.

    Separate architecture task. Do not combine with Stage 01.

Stage 05: ReSTIR PT Enhanced techniques.

    Separate research implementation task. Do not combine with Stage 01.

Minimum Completion Report
-------------------------
Every micro-slice must end with:

- Micro-slice completed.
- Files changed.
- Build command run and result.
- Deployment actions performed.
- Runtime modes/CVars the user should test.
- What the user should compare visually.
- What is explicitly not implemented.
- Recommended next micro-slice.

Do not start the recommended next micro-slice in the same task.
