Task 01: Baseline GI Reservoir Reuse
Date: 2026-05-16
Project: rbdoom-3-bfg-rt path tracing / mode 56 ReSTIR PT

Task Boundary
-------------
Implement the first baseline ReSTIR PT correction for the GI side:

    GI initial reservoir
        -> GI temporal reservoir reuse
        -> GI spatial reservoir reuse
        -> final resolve consumes the completed GI reservoir

The task is successful when the mode-56-derived final resolve no longer shades
the raw GI initial reservoir as the RR-facing GI estimate.

Stop after this task. Do not add RTX Remix enhancements. Do not merge DI and GI
reservoirs. Do not implement ReSTIR PT Enhanced paper features.

Why This Matters
----------------
User testing showed that the GI view contains many black/dark samples on bright
surfaces. Albedo floors and DLSS/RR input brightness changes only turn black
dropout into grey dropout. This points upstream: the reservoir estimate itself
is too sparse or under-reused before it becomes image radiance.

Current branch behavior is split:

- Direct lighting has a staged temporal/spatial reservoir path.
- GI/indirect has an initial-reservoir producer and visible initial views.
- Mode 56 combines direct spatial output with GI initial output.

The GI path is therefore behind the direct path. This task narrows that gap.

Implementation Rules
--------------------
- Use the existing RTXDI PT reservoir ABI already integrated in the branch.
- Use the existing RAB bridge and primary-surface history plumbing.
- Keep existing direct temporal/spatial behavior intact.
- Keep existing modes 53, 54, and 55 useful as GI initial diagnostics.
- Keep mode 56 as the first combined validation path.
- Preserve debug access to the GI initial reservoir. Do not delete the initial
  diagnostic views just because final resolve should stop using them.
- Keep CVar defaults conservative.
- Add pass-plan dumps or CVar names if needed, but do not bury behavior behind
  undocumented switches.

Do Not Do
---------
- Do not copy reference source code into this repository.
- Do not implement RTX Remix sample stealing, adaptive history, RR
  compatibility randomization, virtual samples, or boiling filters in this task.
- Do not implement ReSTIR PT Enhanced duplication maps, reciprocal paired
  spatial reuse, vector-valued shading weights, unified reservoirs, or dual
  motion vectors in this task.
- Do not change mode 18 default rendering.
- Do not remove mode 53-55 diagnostics.
- Do not solve the problem by adding a brightness floor after final shading.

Suggested Slice A: Map Current GI Reservoir Flow
------------------------------------------------
Before editing, write down or keep notes for yourself:

1. Which dispatch writes the GI initial reservoir in mode 56?
2. Which buffer index/page contains the GI initial output?
3. Which shader loads the final GI contribution for mode 56?
4. Which existing helper functions already call RTXDI_PTTemporalResampling and
   RTXDI_PTSpatialResampling for the direct path?
5. Whether those helpers are direct-only because of RAB mode flags or because of
   host pass routing.

If this mapping reveals that the GI temporal/spatial path already exists but is
not routed into mode 56, prefer wiring the existing path over duplicating it.

Suggested Slice B: Add GI Temporal Reservoir Producer
-----------------------------------------------------
Add a GI temporal producer only if no existing route can produce it.

Requirements:

- Consume the GI initial reservoir output.
- Use current primary-surface history for the current surface.
- Use previous-frame reservoir input through the existing RTXDI PT buffer
  indexing scheme.
- Preserve temporal depth/normal/material validation behavior where applicable.
- Keep previous-frame NEE light reuse conservative. Respect the current
  temporal NEE safety rule unless the user explicitly asks to revisit it.
- Store a GI temporal output reservoir in an explicit buffer index/page.
- Add a diagnostic or pass dump that confirms this producer ran.

Stop point if Slice B is commissioned alone:

    Build, deploy, and report. Do not start GI spatial reuse until asked.

Suggested Slice C: Add GI Spatial Reservoir Producer
----------------------------------------------------
Add GI spatial reuse after GI temporal output exists.

Requirements:

- Consume the GI temporal output reservoir.
- Use validated neighboring current-frame reservoirs.
- Keep sample count and radius conservative.
- Reuse existing CVars if they semantically apply, or add GI-specific CVars if
  sharing direct-lighting controls would make testing ambiguous.
- Store a GI spatial output reservoir in an explicit buffer index/page.
- Add source/acceptance diagnostics only if needed to prove the pass is working.

Stop point if Slice C is commissioned alone:

    Build, deploy, and report. Do not add Remix hardening or unified reservoirs.

Suggested Slice D: Final Resolve Consumes Completed GI Reservoir
---------------------------------------------------------------
After temporal/spatial GI output exists, update the mode-56-derived final
resolve:

- Load the completed GI reservoir, not the raw GI initial reservoir.
- Use `TargetFunction * WeightSum` style final contribution for the diagnostic
  estimate.
- Keep final visibility/validation policy explicit.
- Preserve a debug mode or CVar that can compare:
    raw GI initial
    GI temporal
    GI spatial/final
- Keep `PathTraceRRInputColor` fed by the completed GI estimate, not the raw
  initial estimate.

Stop point:

    Build, deploy, and report. Do not continue into RTX Remix or paper features.

Expected Result
---------------
This task may not make the final image clean by itself. The expected improvement
is narrower:

- GI should no longer look like a raw one-sample initial reservoir field.
- Dark/dropout pixels on bright surfaces should reduce if temporal/spatial reuse
  accepts useful neighbors/history.
- Any remaining artifact should be attributable to reuse validation,
  disocclusion, visibility, correlation, or final shading, rather than direct
  raw initial-reservoir exposure.

If the image does not improve, report which stage failed:

- temporal reuse rejects too much,
- spatial reuse rejects too much,
- reservoirs have invalid target/weight data,
- final resolve loads the wrong buffer index,
- visibility invalidates useful samples,
- previous-frame history is unavailable or stale,
- source/path attribution shows poor initial candidates.
