These sketches are handwritten pseudocode for rbdoom-3-bfg-rt agents. They are

Use reference projects to understand concepts, but do not copy/paste licensed

Reference-First Protocol
------------------------
These sketches are not a license to invent new ReSTIR PT math. They are only
branch-local reminders of the shape that must be checked against the local
reference implementations before coding. If a detail is unknown, inspect the
RTXDI / RTX Remix implementation first and translate the existing idea into the
rbdoom pass graph.

Required behavior for future implementation slices:

- Find the matching reference step before editing code.
- Preserve the reference role of current sample, previous sample, neighbor
  sample, target function, Jacobian, normalization, and final shading.
- Do not replace reference-defined processing with ad hoc brightness scaling,
  sample keeping, or reservoir weighting.
- If the reference cannot fit directly, document the engine-specific mismatch
  and the smallest local adaptation needed.

Anti-Pattern From Failed Attempt
--------------------------------
Do not do this:

    RTXDI_PTReservoir temporal = GenerateRestirPTInitialReservoir(surface, pixel);
    StoreTemporalReservoir(pixel, temporal);

That is not temporal reuse. It is a fresh initial sample relabeled as temporal.

Do not do this either:

    Add RestirPTGiReservoirs as a side buffer,
    but keep RTXDI_PTTemporalResampling / RTXDI_PTSpatialResampling reading the
    original RestirPTReservoirs buffer.

That creates two reservoir worlds. The final resolve may read the side buffer,
but RTXDI reuse did not actually operate on that side buffer's history.

Current Local Outlier
---------------------
The current known baseline final resolve has GI behavior shaped like this:

    RTXDI_PTReservoir giReservoir = LoadRestirPTInitialReservoir(pixel);
    float3 giContribution = giReservoir.TargetFunction * giReservoir.WeightSum;

That is acceptable as a baseline and diagnostic. It should not be replaced until
new debug views prove a better GI temporal/spatial output visually.

01B Diagnostic Sketch
---------------------
Goal: add comparison views without changing normal mode 56 output.

Shape only:

    uint RestirGiDebugView()
    {
        return clamp(cvarValue, 0u, 4u);
    }

    float3 EvaluateGiInitialForDebug(uint2 pixel)
    {
        RTXDI_PTReservoir r = LoadRestirPTInitialReservoir(pixel);
        return IsUseful(r) ? r.TargetFunction * max(r.WeightSum, 0.0) : 0.0;
    }

    float4 EvaluateGiDebugView(uint2 pixel)
    {
        if (view == 1)
            return ToneMap(EvaluateGiInitialForDebug(pixel));

        if (view == 2)
            return float4(0, 0, 0, 1); // reserved temporal until 01C

        if (view == 3)
            return float4(0, 0, 0, 1); // reserved spatial until 01D

        if (view == 4)
            return ReservoirValidityColor(LoadRestirPTInitialReservoir(pixel));
    }

Normal mode 56 and `PathTraceRRInputColor` must remain unchanged in 01B.

01C Temporal Proof Sketch
-------------------------
Goal: expose real temporal GI candidate in debug view only.

Required properties:

- It consumes GI initial output.
- It uses a GI-owned previous/current reservoir history route.
- It calls true temporal resampling or a clearly documented branch-local
  equivalent.
- It does not alter normal mode 56 final resolve.

Shape only:

    RTXDI_PTReservoir GenerateGiTemporalReservoir(surface, pixel)
    {
        RTXDI_PTReservoir current = LoadGiInitialReservoir(pixel);

        // Must be true temporal reuse. Do not regenerate current and call it
        // temporal.
        RTXDI_PTReservoir temporal = RunTemporalResamplingOnGiDomain(
            current,
            previousGiHistory,
            motion,
            validation);

        return temporal;
    }

If the implementation cannot make `RunTemporalResamplingOnGiDomain` honest,
stop and report why. Do not fake it.

01D Spatial Proof Sketch
------------------------
Goal: expose GI spatial candidate in debug view only.

Required properties:

- It consumes the user-accepted GI temporal output.
- It combines validated current-frame GI neighbors.
- It keeps initial and temporal debug views available.
- It does not alter normal mode 56 final resolve.

Shape only:

    RTXDI_PTReservoir GenerateGiSpatialReservoir(surface, pixel)
    {
        RTXDI_PTReservoir temporal = LoadGiTemporalReservoir(pixel);
        if (!IsUseful(temporal))
            return temporal;

        RTXDI_PTReservoir spatial = RunSpatialResamplingOnGiDomain(
            temporal,
            currentGiNeighborReservoirs,
            depthNormalMaterialValidation);

        return spatial;
    }

01E Resolve Switch Sketch
-------------------------
Goal: switch only after user visually accepts the GI spatial/temporal debug
output.

Required fallback:

    if (useGiInitialFallback)
        giReservoir = LoadRestirPTInitialReservoir(pixel);
    else
        giReservoir = LoadAcceptedGiFinalReservoir(pixel);

Do not remove the debug comparison views.
