Why This Folder Exists
----------------------

The existing RRX path has produced repeated false positives:

    fallback/base lighting visible under proof views
    synthetic selectors masking real producer failure
    synthetic reservoirs masking current/previous page ownership
    previous-temporal-as-best acting as a fake accumulation path
    display logic creating apparent accumulation from brightness ramps

Do not ask workers to "fix accumulation" in that path.

    RDI-01

Stop after RDI-06 build/deploy for user visual testing.

Review Checklist
----------------

Reject a worker result if it:

    changes math without an explicit approved contract
    uses "looks right" as proof
    hides invalid pixels with albedo/fallback lighting
    enables spatial or best lights before temporal proof
    edits outside the declared write set
    reads or writes data outside io_whitelist.txt
    omits the owned DI initial / NEE-equivalent producer
    cannot identify producer write and consumer read
    cannot explain no-light, no-previous, and zero-target behavior
