Clean-Room DI CVar Contract
===========================

Purpose
-------

The clean-room Remix DI path must have its own CVar namespace.

It must not be hidden behind, overloaded into, or implicitly activated by:

    r_pathTracingDebugMode 56
    r_pathTracingRestirPTDiDebugView
    existing RRX debug CVars
    existing mode-56 fallback/combined preview CVars

Reason:

    The previous RRX work became ambiguous because workers routed new behavior
    through mode 56 and existing debug views. That allowed fallback lighting,
    old ReSTIR PT preview output, and synthetic RRX code to masquerade as DI
    temporal proof.


Required CVar Namespace
-----------------------

All new controls must begin with:

    r_pathTracingCleanRtxdiDi

Workers may not invent another prefix.


Required CVars
--------------

These names are reserved for the first implementation. Workers must use these
exact names unless the manager updates this file.


r_pathTracingCleanRtxdiDiEnable

    Type:
        integer/bool

    Default:
        0

    Meaning:
        Enables the clean-room Remix DI path.

    Rule:
        When 0, the new clean-room path must not run.
        When 1, the clean-room path may run only through its own explicit route.


r_pathTracingCleanRtxdiDiView

    Type:
        integer

    Default:
        0

    Meaning:
        Selects clean-room DI debug output.

    Initial values:
        0 = disabled / no clean-room debug output
        1 = full-screen route sentinel
        2 = primary-surface adapter status
        3 = analytic-light adapter status
        4 = raw flat-diffuse analytic DI, temporal off/current only
        5 = raw flat-diffuse analytic DI, temporal on
        6 = raw flat-diffuse split view, current-only vs temporal
        7 = selected-light identity and reservoir M/history
        8 = reservoir weight/targetPdf and rejection status
        16 = material validation resolve using clean current/temporal/spatial
            DI reservoirs and primary-surface material fields

    Current diagnostic subviews:
        r_pathTracingCleanRtxdiDiView8Band -1 = stacked view-8 bands
        r_pathTracingCleanRtxdiDiView8Band 0..7 = one legacy temporal/PDF band
            full-screen
        r_pathTracingCleanRtxdiDiView8Band 8 = selected-sample black-output
            classifier; with temporal on, left is current and right is temporal

    Stage availability:
        1 is the RDI-01 route sentinel.
        2-8 are reserved contract values. Before the matching staged producer
        exists, they must route to an explicit deferred/not-available clean-room
        status, not to fallback lighting or an existing ReSTIR/RRX debug view.

    Rule:
        Views must write every pixel and may not show fallback lighting.
        Views 4-6 are the only user-facing flat-diffuse accumulation proof
        views. View 16 may show material/texture color for validation and may
        use temporal plus spatial reuse, but is not a replacement for raw
        flat-diffuse temporal proof. Status colors, reservoir bands, dumps,
        buffer survival, or green valid pixels are diagnostics only and must
        not be reported as temporal accumulation.

    Deferred:
        DLSS Ray Reconstruction guide/input writes are not part of view 16.
        The follow-up slice must add clean-owned guide writes explicitly rather
        than using mode 56 or r_pathTracingRestirPTDiDebugView output as proof.


r_pathTracingCleanRtxdiDiTemporal

    Type:
        integer/bool

    Default:
        1

    Meaning:
        Enables the clean-room DI temporal reuse stage.

    Rule:
        When 0, output must be current-only initial reservoir diagnostics.
        When 1, output may use previous-frame reservoir data.


r_pathTracingCleanRtxdiDiInitialVisibility

    Type:
        integer/bool

    Default:
        0

    Meaning:
        Enables RTXDI initial selected-sample visibility in the clean DI
        initial producer.

    Rule:
        This is a diagnostic/parity switch, not a standalone proof of Remix
        temporal behavior. It must not be claimed as a fix unless temporal-on
        output visibly changes from black/noisy invalid samples to stable raw
        flat-diffuse accumulation on real Doom analytic lighting.

    Current finding:
        In rbdoom clean view 12, the switch reaches the shader and is reported
        in r_pathTracingCleanRtxdiDiDump as initialVisibility=1, but it did not
        visibly improve the current clean temporal path. Treat that as evidence
        that the temporal surface/reprojection/visibility consumer still
        deviates from Remix/RRX.


r_pathTracingCleanRtxdiDiSpatial

    Type:
        integer/bool

    Default:
        0

    Meaning:
        Clean-room Remix DI spatial reuse after the temporal producer pass.

    Rule:
        Keep this off for raw flat-diffuse temporal proof views unless a
        spatial slice is explicitly under test. View 16 is material validation
        and may use this CVar as part of the real ReSTIR DI temporal+spatial
        route.


r_pathTracingCleanRtxdiDiBestLights

    Type:
        integer/bool

    Default:
        0

    Meaning:
        Reserved for future Remix best-light feedback.

    Rule:
        For tasks RDI-00 through RDI-06, this must remain forced off. Workers
        must not implement or depend on best-light feedback during the first
        milestone.


r_pathTracingCleanRtxdiDiDenoiser

    Type:
        integer/bool

    Default:
        0

    Meaning:
        Reserved for future denoiser/confidence/gradient integration.

    Rule:
        For tasks RDI-00 through RDI-06, this must remain forced off.


r_pathTracingCleanRtxdiDiFallbackLighting

    Type:
        integer/bool

    Default:
        0

    Meaning:
        Reserved for future beauty integration fallback.

    Rule:
        For proof/debug views, this must remain 0. Invalid or missing reservoir
        data must produce explicit status colors, not albedo, GI, direct light,
        or mode-56 fallback output.


r_pathTracingCleanRtxdiDiLightMode

    Type:
        integer

    Default:
        1

    Initial values:
        0 = no lights, negative-path test
        1 = Doom analytic lights only
        2 = emissive lights only, deferred until after analytic proof
        3 = analytic + emissive, deferred until after both isolated proofs

    Rule:
        RDI-03 through RDI-06 must use value 1 unless explicitly testing the
        no-light negative path with value 0.


r_pathTracingCleanRtxdiDiDump

    Type:
        integer/bool

    Default:
        0

    Meaning:
        Emits one-shot CPU diagnostics for the clean-room DI route.

    Required dump fields:
        enable state
        view id
        temporal/spatial/best-lights/denoiser/fallback states
        light mode
        reservoir page roles
        dispatch dimensions
        active shader route

Historical note:

    r_pathTracingCleanRtxdiDiExternalPdfNeeCurrent was an old analytic PDFNEE
    consume diagnostic. It is not the replacement producer contract, not proof
    for RLU current-frame estimator work, and must not be required by the
    normal clean-room DI path.


Forbidden CVar Coupling
-----------------------

Workers must not require:

    r_pathTracingDebugMode 56
    r_pathTracingRestirPTDiDebugView 72
    r_pathTracingRestirPTRrxSyntheticPrimaryPatch 1
    r_pathTracingRestirPTRrxDebugPreviousTemporalAsBest 1
    r_pathTracingRestirPTMode56Accumulation 1

Workers may use existing global path-tracing infrastructure CVars only when the
task names them, for example build/runtime toggles that already control Vulkan
or path tracing generally.


Runtime Baseline
----------------

The intended clean-room validation baseline is:

    r_pathTracingCleanRtxdiDiEnable 1
    r_pathTracingCleanRtxdiDiView 5
    r_pathTracingCleanRtxdiDiTemporal 1
    r_pathTracingCleanRtxdiDiInitialVisibility 0
    r_pathTracingCleanRtxdiDiSpatial 0
    r_pathTracingCleanRtxdiDiBestLights 0
    r_pathTracingCleanRtxdiDiDenoiser 0
    r_pathTracingCleanRtxdiDiFallbackLighting 0
    r_pathTracingCleanRtxdiDiLightMode 1

Temporal-off comparison:

    r_pathTracingCleanRtxdiDiTemporal 0

Initial-visibility parity check:

    r_pathTracingCleanRtxdiDiInitialVisibility 1
    r_pathTracingCleanRtxdiDiDump 1

Expected dump field:

    initialVisibility=1

This check only proves the switch is wired. It does not prove temporal
accumulation.

No-light negative path:

    r_pathTracingCleanRtxdiDiLightMode 0


Review Rule
-----------

Reject any worker result that activates clean-room DI through mode 56 or an
existing RRX debug view instead of this CVar namespace.
