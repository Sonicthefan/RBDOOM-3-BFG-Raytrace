GI Validation Matrix
====================

Same discipline as the DI matrix: human-visible behavior is the proof.
Counters, bands, and clean builds are diagnostics only. All proofs use a map
location with at least one bright colored surface near a neutral receiver.


RGI-01 scaffold
    enable=0: GPU capture / dump shows zero GI dispatches, zero GI writes
    enable=1 view sentinel: every pixel written from GI lane resources only
    DI output bit-identical with GI enabled vs disabled (sentinel aside)

RGI-02 producer
    view 1: per-pixel noisy radiance; changes every frame (new rays);
        dark on surfaces facing away from all bounce light
    view 2: hit position/normal bands stable for static camera
    secondary decal/transparent card hits: open issue. A broad reject of
        decal/alpha cards in `0e586ea7` caused severe texture destruction and
        repeatable crashes, so do not reapply that approach. The future fix
        needs a receiver-aware decal/alpha contract, not a coarse material-flag
        rejection in the producer.
    dynamic characters/effects: open issue when they appear to seed GI
        emission. Do not solve this by rejecting all dynamic/card materials at
        secondary hit time; validate the emissive inventory/source contract.
    sky/miss pixels: zero radiance, flagged invalid, no NaN view
    firefly cvar 0 vs default: clamp visibly removes outliers only

RGI-03 initial reservoir
    view 3 == view 1 modulo W weighting; no structural differences
    NaN/INF sweep clean (reuse DI dump tooling pattern)

RGI-04 temporal
    static camera: visible convergence over ~1-2 seconds in view 4
    temporal cvar 0: view 4 identical to view 3 (pass-through proof)
    camera strafe: no trailing smear; disocclusion regions show
        single-sample noise that re-converges
    light toggled off (script or trigger): stale energy decays within
        maxReservoirAge frames, does not persist indefinitely
    jacobian cvar 0 vs 1: 1 must not be darker overall (validation must
        reject outliers, not energy)

RGI-05 NEE cache seed
    seed on: lower variance in view 3/4 near cached lights
    seed off/on: no mean-brightness shift on a converged still
    r_pathTracingCleanRestirGiNeeCacheSecondary 0: no cell-correlated strobe
        from secondary-hit cache misses; primary seed testing remains valid
    r_pathTracingCleanRestirGiNeeCacheSecondary 1 with
        r_pathTracingCleanRestirGiNeeCacheSecondaryMode 1: secondary cache
        probes occur only on specular/glossy producer rays with secondary
        roughness <= r_pathTracingCleanRestirGiNeeCacheSecondaryRoughness.
        Rough diffuse surfaces should be much closer to secondary=0 cost.
    r_pathTracingCleanRestirGiNeeCacheSecondaryMode 2: legacy broad opt-in
        for A/B testing; expect higher cost because all secondary producer
        rays may query the cache.
    r_pathTracingCleanRestirGiNeeCacheSecondaryProbability 0.25: visible
        result should move toward the fallback secondary proposal while cache
        probe cost drops. 1 restores the full policy-selected attempt rate.
    view 20: with NEE cache unavailable, dark blue; with seed/provider enabled,
        primary surfaces should show cache providers as green/orange-red,
        fallback-with-candidates as yellow, candidate-record replay failure as
        orange-brown, provider-empty-with-candidates as magenta, shared fallback
        providers with no usable candidates as bright cyan/blue/violet, or
        GI-synthesized no-candidate fallbacks as dark cyan/blue/violet. A
        mostly brown screen means the provider is ready but neither the shared
        provider nor the GI fallback can produce a result for the mapped cells.
        Purple/cyan fault colors mean the provider cannot be replayed at the
        mapped cell and should be treated as a contract bug, not a
        lighting-art difference.

RGI-05B roulette
    r_pathTracingCleanRestirGiContinuationRoulette 1 applies only when
        MaxBounces is 2 and only to the secondary-to-tertiary continuation
        segment. It must not change MaxBounces 1 output.
    roulette min 0.6 / max 0.9: expect lower second-bounce cost and higher
        variance, with no stable darkening/brightening after convergence.
    roulette 0: full-rate second-bounce reference for quality/cost comparison.
    r_pathTracingCleanRestirGiContinuationDirectProbability 0.5: default
        terminal direct-light budget for the tertiary hit. Set to 1 for the
        full-rate reference; lower values should improve FPS with more direct
        lighting variance. Emissive terminal hits should remain visible even
        when this is 0.
    r_pathTracingCleanRestirGiSecondaryDirectProbability 0.5: default
        first-secondary direct-light budget. Set to 1 for the old full-rate
        one-bounce reference; set to 0 to isolate the primary-to-secondary
        material ray cost. Lower values should improve FPS with higher direct
        lighting variance but should not change producer hit geometry.
    r_pathTracingCleanRestirGiContinuationOpaqueTrace 1: default second-bounce
        fast traversal. It should improve MaxBounces 2 cost in alpha-heavy
        scenes while leaving MaxBounces 1 unchanged. Set to 0 to compare the
        full alpha-any-hit continuation reference.

RGI-05C multi-bounce
    r_pathTracingCleanRestirGiMaxBounces 1: baseline one-bounce producer path.
    r_pathTracingCleanRestirGiMaxBounces 2: one additional
        secondary-to-tertiary continuation ray is traced. Expect higher cost,
        brighter/less black indirect in areas that need a second bounce, and
        more variance. The tertiary hit uses terminal emissive/direct lighting
        only; no recursive third bounce.
    specular producer on: continuation may use the same diffuse/specular
        mixture PDF at glossy secondary surfaces. Rough diffuse surfaces should
        mainly behave as diffuse continuation.

RGI-06 spatial
    spatial cvar 0: output identical to temporal output
    spatial on: convergence visibly faster; checker-pattern or blotch
        boiling absent for static camera
    thin-wall scene: no indirect light leaking across occluders

RGI-07 final shading + resolve
    view 6: indirect GI with diffuse + specular; black where no bounce light possible
    view 11/12: split final diffuse/specular lobes; view 11 + view 12 should match view 6 before tonemap
    resolve on: colored wall bleeds onto neutral floor; removing the wall
        (or its light) removes the bleed
    resolve off: combined output identical to pre-GI baseline

RGI-RR-05 reflective debug
    view 13: specular producer radiance; blue means cvar off, red means sampled direction but missed/rejected
    view 14: specular producer hit geometry; alternating position/normal bands
    view 15: specular producer PDF health; red=spec probability, green=mixture PDF, blue=specular PDF
    rough ordinary walls: specular-producer cvar should not cost an extra
        candidate ray; glossy/high-F0 receivers should still show producer hits
    view 16: final specular hit distance bands; red near, green mid, blue far
    view 17: specular producer eligibility/material gate; red only on
        glossy/high-F0 eligible receivers, green is smoothness, blue is F0
    view 18: specular reuse state; red means glossy reuse quarantine, orange
        temporal disabled, yellow no valid history, green accumulated history,
        blue specular producer off, dark green/gray non-eligible or invalid
    view 19: stored split specular output; should match view 12 in RGB, yellow
        marks invalid stored values
    r_pathTracingCleanRestirGiRrHitDistance 1: with specular producer enabled,
        eligible glossy/high-F0 receivers write linear hit distance into
        PathTraceRRGuideHitDistance; rough ordinary walls remain zero
    r_pathTracingCleanRestirGiRrSpecularInput 1: with specular producer enabled,
        eligible glossy/high-F0 receivers add stored specular GI into
        PathTraceRRInputColor; in clean DI views 12/16 with DLSS-RR on and
        GI view 0, this export occurs before DLSS-RR consumes the input color
    r_pathTracingCleanRestirGiRrSpecularInput 0 with resolve on: no specular
        GI should be added to PathTraceRRInputColor by the beauty resolve;
        turning the CVar to 1 should add the specular lobe once, not double it
