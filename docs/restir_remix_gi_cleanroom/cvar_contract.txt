Clean-Room GI CVar Contract
===========================

Purpose
-------

The clean-room Remix GI path gets its own CVar namespace, with the same
isolation rules as the DI lane: it must not be hidden behind, overloaded
into, or implicitly activated by mode 56, RRX debug CVars, or any DI CVar.

The GI path may DEPEND on the DI lane being enabled (it reuses DI-owned
surface history and light sampling at secondary vertices), but a DI CVar must
never silently enable GI.


Required CVar Namespace
-----------------------

All new controls must begin with:

    r_pathTracingCleanRestirGi

Workers may not invent another prefix.


Required CVars
--------------

r_pathTracingCleanRestirGiEnable
    int, default 1.
    When 0 the GI path must not run: no producer dispatch, no GI reservoir
    writes, no GI contribution to resolve. When 1 the path runs only through
    its own explicit route.

r_pathTracingCleanRestirGiView
    int, default 0.
    Reservoir-owned debug view selector, same discipline as the DI views:
        0  off (GI contributes to combined resolve only)
        1  producer radiance (raw initial sample, pre-reservoir)
        2  producer hit geometry (position/normal bands)
        3  initial reservoir radiance * W
        4  temporal output radiance * W
        5  spatial output radiance * W
        6  final shaded indirect GI (diffuse + specular, isolated, no DI)
        7  reservoir M / age diagnostics
        8  route sentinel
        9  secondary material albedo
        10 secondary material texture-source flags
        11 final shaded diffuse lobe
        12 final shaded specular lobe
        13 specular producer radiance
        14 specular producer hit geometry
        15 specular producer PDF health
        16 specular lobe hit distance
        17 specular producer eligibility/material gate
        18 specular reuse state
        19 stored specular output
        20 NEE-cache provider state at the primary surface
    Views must read only GI-lane resources.

r_pathTracingCleanRestirGiTemporal
    int, default 1. Enables temporal resampling. When 0, the initial
    reservoir passes through to the temporal output page unchanged
    (matching Remix's enableReSTIRGITemporalReuse=off behavior).

r_pathTracingCleanRestirGiSpatial
    int, default 1. Enables spatial reuse for the base Remix-shaped GI
    path. When 0, spatial input passes through to the spatial output page
    unchanged for A/B testing.

r_pathTracingCleanRestirGiTemporalBiasCorrection
    int, default 1 (BASIC). Clamped to [0, 1] for baseline; values above
    BASIC are a deferred feature and must clamp down, mirroring the frozen
    stub's min() clamp.

r_pathTracingCleanRestirGiJacobian
    int, default 1. Enables jacobian validation of reused samples.
    Exposed because it is the most likely bias/instability suspect during
    bring-up; turning it off is a diagnostic, not a shipping mode.

r_pathTracingCleanRestirGiMaxHistoryLength
    int, default 4. Temporal history cap matching deployed-game Remix tuning
    (field finding 2026-06-11: the Remix SDK base value of 30 is wildly
    different from what shipped games configure and over-weights stale
    history). 0 bypasses previous-reservoir temporal history.

r_pathTracingCleanRestirGiMaxReservoirAge
    int, default 50. Stale-radiance backstop for the baseline (no
    gradient validation yet).

r_pathTracingCleanRestirGiFireflyThreshold
    float, default matches the producer's clamp policy; 0 disables.
    Applied to initial-sample radiance only, never to reused reservoirs.

r_pathTracingCleanRestirGiMaxBounces
    int, default 1. Producer path depth. 1 is the existing
    primary-to-secondary producer ray with secondary emissive/direct lighting.
    2 adds one secondary-to-tertiary continuation ray, shades the tertiary hit
    with terminal emissive/direct lighting, and folds that through the
    secondary material BSDF/PDF. Values above 2 are clamped until path-depth
    state, roulette, and denoising policy exist.

r_pathTracingCleanRestirGiContinuationRoulette
    int, default 1. Applies Russian roulette only to the
    secondary-to-tertiary continuation segment when MaxBounces is 2. This is
    the valid roulette placement for the current two-segment proof; it does
    not sparsify the first producer ray.

r_pathTracingCleanRestirGiContinuationRouletteMin
    float, default 0.6. Minimum continuation probability for the
    secondary-to-tertiary segment.

r_pathTracingCleanRestirGiContinuationRouletteMax
    float, default 0.9. Maximum continuation probability for the
    secondary-to-tertiary segment. Lowering this trades performance for
    variance by terminating more second-bounce paths.

r_pathTracingCleanRestirGiContinuationDirectProbability
    float, default 0.5. Probability for the terminal direct-light proposal at
    the tertiary hit when MaxBounces is 2. Accepted direct samples are
    compensated by 1 / probability. Set to 1 for a full-rate reference, or
    lower it to reduce second-bounce light/shadow work at the cost of more
    variance. Terminal emissive contribution remains full-rate.

r_pathTracingCleanRestirGiSecondaryDirectProbability
    float, default 1. Probability for the direct-light proposal at the first
    secondary hit. Accepted direct samples are compensated by 1 / probability.
    Set below 1 to reduce one-bounce shadow-ray cost at the cost of producer
    sparsity; set to 0 to measure material-bounce cost without secondary NEE.

r_pathTracingCleanRestirGiSecondaryDirectSamples
    int, default 1. Direct-light proposal count at the first secondary hit.
    Samples are averaged by 1 / count, so increasing this improves producer
    coverage without changing expected energy, at direct shadow-ray cost. Raise
    above 1 only for explicit producer-density diagnostics; the default keeps GI
    on the normal one-path-per-pixel budget.

r_pathTracingCleanRestirGiContinuationOpaqueTrace
    int, default 1. When MaxBounces is 2, traces the secondary-to-tertiary
    continuation ray as opaque so the optional second segment does not force
    alpha any-hit traversal. The first primary-to-secondary producer ray still
    uses the full alpha-capable material path. Set to 0 for the full material
    continuation reference if alpha-cutout geometry needs to affect second
    bounces.

r_pathTracingCleanRestirGiNeeCacheSeed
    int, default 0 for bring-up. When 1, merges the NEE-cache-seeded
    reservoir at the temporal pass's initial-sample step (Remix
    neeCache.enableOnFirstBounce equivalent). Off for the first proofs so
    producer correctness is judged alone. This GI CVar requests the shared
    clean NEE-cache provider; it does not require the clean-DI
    r_pathTracingCleanRtxdiDiNeeCacheProvider diagnostic CVar to be enabled.

r_pathTracingCleanRestirGiNeeCacheSecondary
    int, default 0. When 1, allows the secondary-bounce GI shader to query
    the shared NEE-cache provider at the secondary hit. This is a master
    switch; the actual after-first-bounce policy is controlled by the mode,
    roughness, and probability CVars below. Off by default because the current
    cache producer is primary-surface populated; keep the primary seed path
    controlled by r_pathTracingCleanRestirGiNeeCacheSeed.

r_pathTracingCleanRestirGiNeeCacheSecondaryMode
    int, default 1. Remix-shaped after-first-bounce cache policy:
        0  disabled even when the secondary master switch is on
        1  specular/glossy producer rays only
        2  all secondary producer rays (legacy broad opt-in behavior)
    Mode 1 is the default because the local dxvk-remix reference gates higher
    bounce NEE-cache use away from diffuse and rough specular rays for
    performance.

r_pathTracingCleanRestirGiNeeCacheSecondaryRoughness
    float, default 0.1. Maximum secondary material roughness for mode 1.
    Lower values reduce cache probes on broad/rough surfaces; higher values
    move toward mode 2 behavior.

r_pathTracingCleanRestirGiNeeCacheSecondaryProbability
    float, default 1. Bernoulli attempt probability after mode/roughness
    gating. This is a practical performance/quality throttle: lowering it
    makes more rays fall through to the existing secondary proposal instead
    of querying the cache.

r_pathTracingCleanRestirGiSpecularProducer
    int, default 0. When 1, traces a cvar-gated glossy/specular initial
    producer ray and merges it into the INIT reservoir before the normal raw
    diffuse producer sample joins the initial RIS update. The existing producer
    radiance/hit debug textures still show the diffuse producer. Views 13-15
    debug the specular producer. View 17 is a no-extra-ray material gate view:
    red means the receiver is eligible for the specular producer, green is
    smoothness (1 - roughness), and blue is F0 luminance scaled for display.
    View 18 is a no-extra-ray reuse state view for reflective receivers: red
    means mirror/glossy reuse quarantine, orange means temporal disabled, yellow
    means no valid temporal history, green means history is accumulating, blue
    means the specular producer is disabled, and dark green/gray are non-eligible
    or invalid surfaces.
    View 19 reads the stored CleanRestirGiIndirectSpecularLobe UAV after final
    shading writes it. Its RGB should match view 12, modulo normal debug
    tonemapping/order, and yellow marks invalid stored values.
    View 20 maps the primary surface into the NEE-cache provider grid and
    colors the provider state/source: dark blue means provider unavailable,
    dark gray invalid primary surface, purple invalid cell, brown valid cell
    with no provider, cyan bad replay contract, green cache-emissive,
    orange/red cache-analytic, bright cyan/blue/violet mean shared-provider
    fallback typed-emissive/typed-analytic/full-RLU with no usable candidates
    in that cell, yellow means the shared provider chose fallback even though
    usable cache candidates exist, orange-brown means candidate records exist
    but none replayed as usable, magenta means GI synthesized fallback because
    the provider slot was empty even though usable candidates exist, and dark
    cyan/blue/violet mean GI synthesized fallback with no usable candidates.
    Selecting view 20 also requests the shared clean NEE-cache provider. It
    reads provider/candidate state for the mapped primary cell; it is not the
    shaded GI seed consume path and must not be used as proof that GI should
    consume PathTraceNeeCacheProviderResults[cell] directly.

r_pathTracingCleanRestirGiRrHitDistance
    int, default 0. When 1, and only when the specular producer is also enabled,
    writes eligible reflective GI specular lobe hit distance into
    PathTraceRRGuideHitDistance. The value is linear world/engine distance, not
    debug-scaled color. Eligibility uses the same reflective-receiver gate as
    the specular producer so rough ordinary walls do not export hit distance.

r_pathTracingCleanRestirGiRrSpecularInput
    int, default 0. When 1, and only when the specular producer is also enabled,
    adds eligible stored specular GI into PathTraceRRInputColor for RR/DLSS
    experiments. This is an opt-in export path: it does not change the normal
    beauty resolve and uses the same reflective receiver/lobe eligibility as
    the hit-distance route. In clean DI views 12/16 with DLSS-RR enabled and
    GI view 0, this export dispatch runs before DLSS-RR evaluates so RR can
    consume the modified input color. This is the only normal GI path that may
    add GI radiance to PathTraceRRInputColor; the beauty resolve must not also
    write RR input or the specular lobe is double-counted.

r_pathTracingCleanRestirGiResolve
    int, default 0 until RGI-07. When 1, the final-shaded GI output is
    added into the combined beauty resolve. It must not write
    PathTraceRRInputColor; RR input is owned by
    r_pathTracingCleanRestirGiRrSpecularInput.


Rules
-----

    No GI CVar may change DI behavior.
    Defaults must produce the deferred-feature-off Remix baseline.
    Every dispatch-gating CVar must be checked on the C++ side, not only
        in shader constants.
    GUI exposure follows the DI lane's existing panel pattern.
