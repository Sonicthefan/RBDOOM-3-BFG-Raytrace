Worker Tasks
============

Use these tasks in order. Do not skip ahead. Each task's required proof is in
validation_matrix.txt. Worker protocol is inherited from
../restir_remix_di_cleanroom/worker_protocol.txt.


RGI-00: GI Contract Audit
-------------------------

Type:
    read-only audit

Goal:
    Verify remix_gi_contract.txt against the reference files and the current
    rbdoom DI lane, and produce the GI io whitelist.

Reference files:
    all files listed in remix_gi_contract.txt

rbdoom files to inspect:
    neo/shaders/builtin/pathtracing/remix_restir_gi/restir_gi_temporal_reuse.rt.hlsl
    neo/shaders/builtin/pathtracing/remix_bridge/RAB_GI*.hlsli
    neo/shaders/builtin/pathtracing/cleanroom_rtxdi/ (current functional DI lane)
    neo/renderer/NVRHI/PathTraceSmokeDispatch.cpp (clean DI dispatch route)
    neo/renderer/NVRHI/PathTraceNeeCache.cpp
    docs/restir_remix_di_cleanroom/io_whitelist.txt

Required output:
    io_whitelist.txt for the GI lane (allowed inputs/outputs per pass)
    confirmation that upstream Rtxdi/GI headers compile-include cleanly from
        RTXDI_RUNTIME_DIR (CMake include registration may be missing; list
        the exact CMakeLists edits needed)
    list of DI-lane symbols the GI lane will link against (surface supplier,
        visibility, light sampling at secondary vertex, NEE cache query)
    any contract corrections to remix_gi_contract.txt

Forbidden:
    no edits outside this docs folder


RGI-01: GI Lane Scaffold + Sentinel
-----------------------------------

Type:
    narrow implementation

Goal:
    Stand up the isolated GI route: CVars, C++ dispatch, GI reservoir pages,
    producer output textures, and a sentinel debug view that writes every
    pixel from the GI lane only.

Allowed write set:
    new shader file(s) under neo/shaders/builtin/pathtracing/remix_restir_gi/
    new C++ file(s) under neo/renderer/NVRHI/
    neo/renderer/NVRHI/PathTraceCVars.cpp / .h
    neo/shaders/CMakeLists.txt, neo/CMakeLists.txt (GI header include reg.)

Requirements:
    r_pathTracingCleanRestirGiEnable / ...View control the route
    GI reservoir buffer with the four pages from RAB_GIReservoirBridge
    producer radiance (rgba16f) + hit geometry textures allocated
    sentinel writes every pixel; no albedo, no lighting, no DI reads
    enable=0 provably dispatches nothing

Forbidden:
    no reservoir math, no rays, no edits to DI behavior


RGI-02: Initial-Sample Producer
-------------------------------

Type:
    implementation

Goal:
    From each valid primary surface, BSDF-sample one indirect ray, trace it,
    shade the hit with the existing secondary-vertex light machinery
    (analytic lights + emissive + NEE cache), and write producer textures.

Contract points (from remix_gi_contract.txt):
    radiance excludes primary BSDF/albedo (incoming radiance only)
    alpha channel = indirect path length
    hit geometry texture stores hit position + normal (portalID slot
        reserved, always invalid for baseline)
    miss = zero radiance, invalid hit geometry flag
    firefly clamp per cvar applies here only

Proof views:
    view 1 (producer radiance), view 2 (hit geometry)

Forbidden:
    no reservoir construction yet
    no multibounce beyond the single indirect vertex's NEE estimate
    no reading DI reservoir buffers
