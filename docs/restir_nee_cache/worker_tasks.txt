NEECACHE-08: Importance / RIS Proposal Selection
------------------------------------------------

Goal:

    Turn the sourcePolicy 2 PDFNEE consume bridge into a real current-frame
    RIS proposal source over the cache/fallback distribution instead of a
    single light draw from the mapped cache cell.

Reference file-equivalent:

    E:/prog/references/RTXDI-main/Libraries/Rtxdi/Include/Rtxdi/DI/InitialSampling.hlsli
    E:/prog/references/RTXDI-main/Libraries/Rtxdi/Include/Rtxdi/LightSampling/PresamplingFunctions.hlsli

Allowed rbdoom write set:

    neo/renderer/NVRHI/PathTraceSmokeDispatch.cpp
    neo/shaders/builtin/pathtracing/pathtrace_smoke.rt.hlsl
    docs/restir_nee_cache

Required:

    cache build updates persistent candidate slots instead of clearing and
        rewriting a deterministic slot set per visible cell
    slot replacement uses bounded RIS over the configured source domain
    slot admission uses stable dense-identity target slots, duplicate refresh,
        empty-slot fill, and materially-stronger target replacement
    cache reads reject hash-cell collisions by comparing stored cell hash with
        the currently mapped world cell
    sourcePolicy 2 streams multiple proposals per pixel
    proposal count comes from r_pathTracingRestirPdfNeeVerifierSamples
    cache candidates are sampled by current importance weight
    fallback proposals remain in the same mixture
    every streamed proposal carries its exact sourcePdf/invSourcePdf
    RTXDI_StreamSample / RTXDI_FinalizeResampling do the RIS selection
    selected identity remains dense current RLU index
    RAB replay boundary remains unchanged
    temporal and spatial remain off

Acceptance:

    build/deploy succeeds
    PDFNEE dump reports sourcePolicy=2(nee-cache-provider)
    PDFNEE dump reports producerHelperSequence contains nee-cache-ris
    changing r_pathTracingRestirPdfNeeVerifierSamples changes proposal quality
        without changing the source domain contract
