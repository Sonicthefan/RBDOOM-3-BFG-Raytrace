Remix-Style NEE Cache CVar Contract
===================================

Purpose
-------

Reserve a narrow CVar surface for the NEE cache and related spatial proposal
lanes.

Default-functionality rule:

    r_pathTracingNeeCacheEnable 1 must build/update the normal cache/provider
    data with useful defaults. DebugView, dump, forced mode, manual parameters,
    and validation views are diagnostics only. They must not be required before
    cells populate or candidates replay through RAB.

Consumer roles:

    GI / secondary NEE consumes the Remix-like NEE cache.
    Clean RTXDI DI may consume a separate lightweight spatial proposal provider
        over dense current RLU identities.
    PDFNEE sourcePolicy 2 is preserved as an alternate visual testing mode.


Required CVars
--------------

r_pathTracingNeeCacheEnable

    0 = disabled
    1 = build/update the normal NEE cache / spatial proposal data

    Default: 0


r_pathTracingNeeCacheMode

    0 = disabled
    1 = Remix-style log/hash learned cache
    2 = reserved for later RTXDI-style onion/full ReGIR escape hatch
    3 = bounded regular grid diagnostic

    Default: 1


r_pathTracingNeeCacheDebugView

    0 = disabled
    1 = route/status sentinel
    2 = cell id
    3 = cell occupancy
    4 = task accumulation map
    5 = emissive triangle candidate map
    6 = analytic candidate identity map
    7 = sourcePdf / invSourcePdf
    8 = cache/fallback mixture source
    9 = empty/fallback reason
    10 = RAB replay validity
    11 = flat consumed-candidate render
    12 = flat full current-RLU render

    Default: 0

    This CVar selects diagnostics only. It must not be needed for normal cache
    build/update when r_pathTracingNeeCacheEnable is 1.

    View 11 renders flat material response from the cache candidate records by
    replaying dense current RLU indexes through the active RLU/RAB load, sample,
    target-PDF, and reflected-BSDF helpers.

    View 12 renders flat material response from the full current dense RLU typed
    ranges through the same RLU/RAB replay path, bypassing the cache candidate
    slot filter. Use it as the control when view 11 misses most lights.

    Both flat views are for inspecting payload/candidate stability only: no
    textures, no visibility proof, no temporal/spatial reuse, no best-light
    feedback, and no final direct-light cache contribution.


r_pathTracingNeeCacheCellResolution

    Remix-style cell resolution parameter. Higher values mean smaller cells.

    Default: 8

    Calibration:

        debugView 2 visual comparison matched the Remix reference when using
        cellResolution 8 with minRange 256. A 64-cell resolution made the
        visible cells about one quarter of the desired width.


r_pathTracingNeeCacheMinRange

    Minimum range for lowest-level cells in Doom world units.

    Default: implementation must choose and document a conservative value.


r_pathTracingNeeCacheCellCount

    Fixed physical cache cell count for hash/log mode.

    Default: implementation must choose and document a fixed budget.


r_pathTracingNeeCacheCandidateSlots

    Candidate slots per cell for each active list, or the total slot budget if
    the implementation stores a packed mixed list.

    Default: implementation must choose and document a fixed budget.


r_pathTracingNeeCacheTaskSlots

    Task slots per cell.

    Default: implementation must choose and document a fixed budget.


r_pathTracingNeeCacheFallbackProbability

    Probability of sampling from the source-domain fallback instead of the
    cache.

    Default: greater than 0.

    A value of 0 is diagnostic only and must be reported as biased/cache-only.


r_pathTracingNeeCacheSourceDomain

    0 = all current RLU lights (diagnostic)
    1 = emissive range only
    2 = analytic range only
    3 = typed cache/fallback range mixture

    Default: 3.

    The shader must consume this field. Domain 0 samples fallback identities
    uniformly from the full dense current RLU and is kept as a noisy diagnostic
    source. Domains 1 and 2 restrict fallback identities to the current typed
    emissive or analytic range. Domain 3 keeps the default provider away from
    the full-RLU uniform fallback; domain 0 remains available when the older
    full dense current-RLU proposal is needed for diagnostics.


r_pathTracingNeeCacheDump

    0 = disabled
    1 = one-shot dump

    Required dump fields:

        enable
        mode
        debugView
        cell count
        task slots
        candidate slots
        RLU current count
        typed ranges
        fallback probability
        cache probability
        selected provider
        first missing contract


PDFNEE Consume CVar
-------------------

This lane is preserved for visual testing and research comparison. It is not
the default Remix-style NEE cache consumer and must not be deleted.
It must not be promoted to the production primary-DI route on the assumption
that RTX Remix or ReSTIR PT already use the NEE cache that way.

r_pathTracingRestirPdfNeeVerifierSourcePolicy

    Existing PDFNEE current-producer source-policy CVar. NEECACHE-07 adds only
    this value:

        2 = consume PathTraceNeeCacheProviderResults as the current-frame
            proposal source

    Source policy 2 requires r_pathTracingNeeCacheEnable 1 and a ready provider
    result buffer for the frame. When unavailable, dumps must report
    firstMissingContract=nee-cache-provider-not-ready-neecache-07 instead of
    silently falling back to typed RLU sampling.


Future DI Proposal CVar
-----------------------

The clean DI spatial proposal provider should have its own opt-in CVar rather
than reusing PDFNEE sourcePolicy.

r_pathTracingCleanRtxdiDiNeeCacheProvider

    0 = clean RTXDI DI uses its current proposal path
    1 = clean RTXDI DI initial sampling may draw proposals from the spatial
        provider / fallback mixture

This CVar must not change clean RTXDI temporal accumulation settings. The
provider may only affect initial current-frame proposal distribution.
When this CVar is enabled with r_pathTracingNeeCacheEnable 1, the clean DI
route must dispatch the NEE-cache candidate build prepass before the first clean
sentinel consume of t74/t75/t77, then may hold that provider snapshot while the
provider remains enabled. Provider readiness must not depend on debugView,
band-10 visual refresh, or any other diagnostic CVar. Disabling the provider
must release the provider snapshot hold. Entering clean-DI view 8 band 10 also
releases the provider snapshot hold because that diagnostic owns the shared
cache buffers while its visual snapshot is active. Leaving clean-DI view 8 band
10 must force-clear the diagnostic-owned cache buffers before view 12 can hold
or consume a provider snapshot again. Any frame that begins with a pending NEE
cache clear, including first allocation/load and the first frame after leaving
band 10, must defer the clean provider build and use the existing clean initial
proposal path for that frame.

Required companion state:

    r_pathTracingCleanRtxdiDiEnable 1
    r_pathTracingCleanRtxdiDiView 12
    r_pathTracingCleanRtxdiDiLightMode 1
    r_pathTracingRemixLightUniverseEnable 1
    r_pathTracingNeeCacheEnable 1

Fallback-only A/B:

    r_pathTracingNeeCacheFallbackProbability 1


GI / Secondary NEE CVars
------------------------

r_pathTracingNeeCacheSecondaryEnable

    0 = path/secondary NEE cache consume disabled
    1 = path/secondary NEE may sample cache candidates according to the bounce
        policy

Default:

    0

r_pathTracingNeeCacheSecondaryDump

    0 = no dump
    1 = print the NEECACHE-10 secondary consumer contract once, then reset to 0

Default:

    0

r_pathTracingNeeCacheSecondaryVisualRefresh

    0 = clean-DI view 8 band 10 reads the existing NEE cache buffers
    1 = refresh the diagnostic candidate field once from the clean route, hold
        that visual snapshot while clean-DI view 8 band 10 remains active, then
        reset to 0
    2 = refresh the diagnostic candidate field every frame

Default:

    0

    The refresh path dispatches the NEE-cache candidate builder from the clean
    route and can be expensive in portal-heavy views. Keep it off for normal
    inspection; use 1 only when the band 10 view needs a fresh cache snapshot.
    The held snapshot is diagnostic-only and is released when leaving band 10,
    so normal provider invalidation resumes for production routes.

This is the default architectural destination for the NEE cache. It must be
testable independently from clean RTXDI DI and from PDFNEE sourcePolicy 2.

First production consume slice:

    candidate class is restricted to current-RLU emissive candidates
    replay goes through RAB_LoadLightInfo / RAB_SamplePolymorphicLight
    contribution uses sourcePdf * solidAnglePdf and existing RAB MIS input
    empty/stale/non-emissive candidates fall back to the existing secondary NEE
        path
    primary DI and clean RTXDI DI CVars are not required for this consumer

Dedicated diagnostic view:

    r_pathTracingCleanRtxdiDiEnable 1
    r_pathTracingCleanRtxdiDiView 8
    r_pathTracingCleanRtxdiDiView8Band 10
    r_pathTracingNeeCacheSecondaryVisualRefresh 1

    Band 10 is a clean-DI route diagnostic for the NEECACHE-10 candidate field.
    It does not consume or modify clean DI reservoirs as secondary lighting.
    It maps the visible primary surface to the NEE cache cell and shows the
    current-RLU emissive candidates the secondary consumer can replay:

        left third = valid emissive candidate occupancy for the cell
        middle third = deterministic selected dense current RLU identity color
        right third = selected sourcePdf and candidate weight scale

    This is the only dedicated visualizer for the first NEECACHE-10 slice.
    Legacy mode 18 and ReSTIR PT debug modes 53-55 are not valid NEE-cache
    secondary proof views.


Future / Not First Slice
------------------------

Do not add until standalone debug proof passes:

    temporal feedback toggle
    spatial reuse toggle
    best-light feedback toggle
    denoiser/confidence/gradient toggle


Forbidden CVar Behavior
-----------------------

NEE cache CVars must not:

    require a multi-CVar recipe for normal functionality
    make debug views the only path where cells populate
    route through mode 56
    enable clean RTXDI temporal consume
    enable RRX reservoir pages
    enable spatial reuse
    enable best-light feedback
    make PDFNEE sourcePolicy 2 the only normal consume path
    use PDFNEE sourcePolicy 2 as proof of Remix/ReSTIR primary-DI parity
    make clean DI consume a cell-owned finished reservoir
    make all pixels in one cache cell consume one deterministic light
    suppress broad lights
    switch to fallback beauty or albedo
    hide effective parameter overrides from the dump
