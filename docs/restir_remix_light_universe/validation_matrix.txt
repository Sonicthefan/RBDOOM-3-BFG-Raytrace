Remix-Shaped Light Universe Validation Matrix
============================================

Rung V-00: Read-Only Contract Audit
-----------------------------------

Purpose:

    Compare the current rbdoom light-universe/remap producer against Remix and
    RTXDI before implementation.

Pass:

    worker names exact reference files, rbdoom read set, proposed write set,
    whitelist ids, CVars, and first implementation boundary

Fail:

    worker starts coding before proving the boundary


Rung V-01: Dense Domain CPU Shell
---------------------------------

Purpose:

    Prove the new lane can own dense current/previous arrays and two-way maps
    without affecting active lighting.

Expected:

    CVars exist
    dumps report dense current/previous counts
    disabled state writes no active route
    map/render-world reset clears previous state

Pass:

    enabling and disabling changes only this lane's resources and dumps

Fail:

    clean RTXDI, PDFNEE, RRX, or mode 56 behavior changes to make the shell pass


Rung V-02: Doom Analytic Dense Mapping
--------------------------------------

Purpose:

    Build dense current/previous Doom analytic domains and two-way maps from
    uploaded real Doom analytic payload identity.

Expected:

    stable room lights map both directions while stationary
    animated payload changes do not break mapping
    spawned/despawned effect lights become current-only or previous-only

Pass:

    dump and debug views agree that stable lights remain mapped while effects
    fail locally

Fail:

    one effect light invalidates unrelated room-light mappings
    dense previous payloads are indexed by stale candidate slots


Rung V-03: Clean RTXDI DI Opt-In A/B
------------------------------------

Purpose:

    Let clean RTXDI view 12 consume the new Remix-shaped analytic domain as an
    opt-in A/B against the old producer and bypass.

Expected:

    old path is legacy/debug only and is not proof or replacement fallback
    bypass remains diagnostic
    new path routes current/previous/remap through the Remix-shaped producer

Pass:

    user sees real Doom analytic temporal-on raw flat-diffuse output become
    visibly less noisy than temporal-off around the same mean lighting

Fail:

    success is claimed from synthetic view 10/11, mirror behavior, status
    colors, or non-crashing builds


Rung V-04: Animated Payload And Effect-Light Diagnostics
--------------------------------------------------------

Purpose:

    Prove animated payloads and spawned/despawned lights are local events.

Expected:

    payload-only signature changes while mapping signature remains stable for
    stable lights
    effect lights show current-only/previous-only classification

Pass:

    room lights continue mapping while sparks or robot lights enter/exit

Fail:

    broad suppression is required
    payload animation is treated as structural remap failure


Rung V-05: Emissive Domain Addition
-----------------------------------

Purpose:

    Add emissive lights only after analytic temporal proof.

Expected:

    final merged emissive list is used
    dense current/previous emissive payloads map by stable emissive identity
    analytic proof math is not changed

Pass:

    emissive-only debug views map coherently, and analytic temporal proof still
    works unchanged

Fail:

    emissive overlap failure changes analytic proof math


Rung V-06: Unified RAB Bridge
-----------------------------

Purpose:

    Expose the new dense domain through RAB runtime loading and translation.

Expected:

    RAB_LoadLightInfo and RAB_TranslateLightIndex consume the new buffers
    temporal consumers no longer inspect old split-domain candidate slots

Pass:

    shader temporal path sees one Remix-shaped domain with typed ranges

Fail:

    shader still reloads legacy split payload arrays through payloadSourceIndex
    for the active proof path


Rung V-07: Active Buffer Ownership
----------------------------------

Purpose:

    Remove ambiguity about which shader buffers are the active polymorphic RLU
    domain.

Expected:

    t66 is documented as the active current RLU payload buffer
    t67 is documented as the active previous RLU payload buffer
    t59/t60 are documented as legacy/debug unless explicitly owned
    NEE cache and ReGIR are documented as proposal experiments, not replacement
        light universes

Pass:

    worker can name the exact RLU producer-to-consumer path without mentioning
    private Doom analytic reloads as the normal path

Fail:

    clean DI, PDFNEE, NEE-cache, or ReGIR tasks are allowed to create a separate
    normal light universe


Rung V-08: Payload Replay Hardening
-----------------------------------

Purpose:

    Prove that active RLU payloads are valid before and after shader upload.

Expected:

    current and previous analytics expose valid radius, influenceRadius, area,
    luminance, sourceWeight, flags, sourceIndex, and identity
    current and previous emissives expose valid area, radiance/luminance,
    sourceWeight, material/primitive identity, and previous remap
    shader debug replay agrees with CPU dump for representative mapped lights

Pass:

    moving near animated lights does not poison unrelated lights, and failures
    are local to the failed light or failed remap

Fail:

    a shader-visible payload differs from the CPU payload and the task proceeds
    to sampling changes anyway


Rung V-09: Typed Range Metadata
-------------------------------

Purpose:

    Prove that RLU typed ranges and bounded sample counts are authoritative.

Expected:

    dumps report emissive and Doom analytic offset/count pairs
    dumps report bounded sample counts for each type
    shader helpers expose the same values
    zero-count ranges are handled without fallback domains

Pass:

    sourcePdf for a selected dense RLU light can be derived from typed range
    metadata and the selected proposal policy

Fail:

    a consumer scans payload records to rediscover the normal range contract
    or uses NEE-cache/ReGIR ownership as the source of range truth


Rung V-10: Clean DI Range Provider
----------------------------------

Purpose:

    Add clean DI shader helpers for RLU typed ranges without changing output.

Expected:

    helpers return active range offset/count by light class
    helpers return bounded sample count by light class
    helpers expose non-empty range count and proposal probability inputs

Pass:

    build succeeds and clean RTXDI view 12 output remains unchanged until typed
    sampling is enabled

Fail:

    helper addition changes reservoir selection, temporal reuse, or final
    lighting before RLU-12


Rung V-11: Clean DI Typed Initial Sampling
------------------------------------------

Purpose:

    Improve clean RTXDI DI current proposals using active RLU typed ranges.

Expected:

    emissive and analytic typed ranges are sampled when present
    sourcePdf matches the actual typed proposal policy
    targetPdf and solidAnglePdf come from RAB replay
    reservoir finalization stores M = 1

Pass:

    clean RTXDI view 12 has the same mean as the RLU baseline, and changing
    candidate/sample counts changes noise predictably

Fail:

    visibility is part of targetPdf
    selected identity is not a dense current RLU index
    NEE-cache/ReGIR is required for normal clean DI function


Rung V-12: Previous-Best Initial Sampling
-----------------------------------------

Purpose:

    Seed clean DI initial sampling with a translated previous-best light after
    typed RLU sampling is correct.

Expected:

    previous best is resolved before temporal reuse
    previous-domain light is translated to current domain with RAB
    translated current light is sampled as a single-light candidate
    RTXDI reservoir helper math combines it with random typed proposals

Pass:

    movement stability improves in overlapping-light areas without regressing
    stationary lighting or animated-light isolation

Fail:

    previous-domain light indices are used directly in the current domain
    temporal reuse is rewritten to hide bad current proposals
    a previous temporal reservoir source is called full Remix best-light parity


Rung V-13 / RLU-14: Emissive Triangle Replay
--------------------------------------------

Purpose:

    Correct clean DI/RLU emissive consume so emissive surfaces behave like real
    triangle area lights instead of center-position point proxies.
    This is the next task after RLU-13 and corrects the non-viable emissive
    point/center conversion path.

Expected:

    each sampled RLU emissive resolves to actual triangle positions and UVs
    RAB emissive sampling uses barycentric triangle sampling
    emissive texture/material evaluation uses sampled triangle UVs when
        available
    areaPdf and solidAnglePdf match the sampled triangle geometry
    failed triangle replay rejects only the failed emissive candidate

Pass:

    clean RTXDI view 12 emissive contribution follows visible emitting
    surfaces, no point-light hotspots appear at emissive centers, and the
    center-fallback diagnostic count is zero for the normal path

Fail:

    emissive surfaces collapse to point lights
    missing triangle geometry is accepted as a fake center/area light
    emissive replay fixes break analytic RLU sampling or current/previous maps
    weighted emissive proposal changes are made before replay correctness is
        proven


Rung V-14 / RLU-15: Clean DI Basic Spatial Reuse
------------------------------------------------

Purpose:

    Add a bounded basic spatial reuse pass to the default clean DI view-12 RLU
    lane after emissive triangle replay is correct.

Expected:

    temporal producer still runs through the existing clean DI sentinel shader
    spatial reuse runs through a separate clean DI spatial shader
    neighbor reservoirs are read from the copied temporal/source page
    spatial output writes a separate spatial reservoir page
    the RLU current/previous dense domains and emissive replay eligibility do
        not change

Pass:

    default clean DI view 12 dump reports spatial=1, selectedCleanShader=2,
    cleanSpatialReservoir=1, pages spatial=u72, and spatialParams with the
    configured samples/disocclusion/radius values

Fail:

    the route uses the sentinel shader for the spatial pass
    spatial reuse changes RLU dense-domain counts, remaps, or emissive replay
    spatial proof depends on denoiser, gradient/confidence, ReGIR, NEE-cache,
        PDFNEE, full-map/full-domain brute force, or mode 56


Hard Stop
---------

Stop at the first failing rung.

Do not continue to denoiser, gradient/confidence, enhanced spatial replacement,
ReGIR-as-default-DI, PDFNEE-as-proof, or broad light suppression to hide a
failing RLU or clean DI sampling contract.  Previous-best initial sampling is
allowed only at V-12 after the active RLU payload, range, and typed proposal
contracts pass.  Weighted emissive proposal work is allowed only after V-13
proves real triangle replay.  Basic spatial reuse is allowed only at V-14 after
the active clean DI emissive replay path passes.
