RLU-07 RTX Portal / Remix Debug Crosswalk
=========================================

Task
----

RLU-07 is a documentation-only comparison pass. It maps rbdoom's current
Remix-shaped light-universe diagnostics to the RTX Remix / RTX Portal concepts
that are useful for side-by-side debugging.

No implementation, build, CVar change, or shader edit belongs to this task.

Producer / Binding Crosswalk
----------------------------

RTX Remix:

    current polymorphic light buffer
    previous polymorphic light buffer
    light mapping buffer
    typed range metadata exposed through the application bridge

rbdoom RLU:

    current polymorphic RLU payload:
        PathTraceRestirLightManagerCurrentPayload
        CleanRtxdiDiRluCurrentLights
        register(t66)

    previous polymorphic RLU payload:
        PathTraceRestirLightManagerPreviousPayload
        CleanRtxdiDiRluPreviousLights
        register(t67)

    current-to-previous map:
        CleanRtxdiDiRluCurrentToPrevious / active RAB light-manager map

    previous-to-current map:
        CleanRtxdiDiRluPreviousToCurrent / active RAB light-manager map

    typed range metadata:
        RestirLightManagerRangeInfo.xy = emissive dense offset/count
        RestirLightManagerRangeInfo.zw = Doom analytic dense offset/count

    bounded proposal metadata:
        RestirLightManagerSampleInfo.x = bounded emissive sample count
        RestirLightManagerSampleInfo.y = bounded Doom analytic sample count
        RestirLightManagerSampleInfo.z = total bounded proposal count
        RestirLightManagerSampleInfo.w = non-empty typed range count


RAB Crosswalk
-------------

RTXDI / Remix bridge concept:

    load current light by dense light index
    load previous light by dense light index
    translate current-domain selected light to previous domain
    translate previous-domain selected light to current domain
    sample polymorphic light through the bridge
    ask the bridge for target PDF and reflected BSDF radiance

rbdoom active RLU bridge:

    RAB_LoadActiveRrxLightInfo(index, false)
        loads current dense RLU payload from t66

    RAB_LoadActiveRrxLightInfo(index, true)
        loads previous dense RLU payload from t67

    RAB_TranslateActiveRrxLightIndex(index, true)
    RAB_TranslateLightIndex(index, true)
        maps current dense index to previous dense index

    RAB_TranslateActiveRrxLightIndex(index, false)
    RAB_TranslateLightIndex(index, false)
        maps previous dense index to current dense index

    RAB_SampleActiveRrxPolymorphicLight(...)
    RAB_SamplePolymorphicLight(...)
        samples a RAB-loaded current or previous RLU light

    RAB_GetLightSampleTargetPdfForSurface(...)
    RAB_GetReflectedBsdfRadianceForSurface(...)
        provide target-PDF and reflected-radiance inputs after replay

Debug Equivalence Limits
------------------------

Comparable now:

    dense current/previous light ownership
    current-to-previous and previous-to-current translation
    typed range offsets and counts
    active RAB replay of selected dense light identity
    payload-only versus structural/mapping signature changes
    local current-only and previous-only events

Not comparable yet:

    Remix previous-best initial-sampling feedback
    Remix denoiser/confidence/gradient behavior
    spatial reuse behavior
    RTX Portal-specific portal-light policy
    final quality of NEE-cache or ReGIR proposal experiments

Explicit gap:

    rbdoom does not yet have a full Remix-style previous-best feedback source
    wired into clean DI initial sampling. Any later approximation that uses a
    previous temporal reservoir selected light must be labeled as an
    approximation and must still translate previous-domain light identity
    through RAB before sampling it in the current domain.


Validation Use
--------------

Use this file to compare debug data, not to claim visual correctness by itself.

Required visual proof still comes from clean RTXDI DI view 12 with:

    real Doom analytic lights
    temporal off = current-only noisy output
    temporal on = visibly less noisy around the same mean
    no spatial reuse
    no best-light feedback for early RLU proof
    no denoiser/confidence/gradient proof
    no fallback beauty/albedo/status-color proof
