Remix-Shaped Light Universe Contract Matrix
==========================================

Goal
----

Build a trustworthy RTXDI-facing light universe for rbdoom that follows the
RTX Remix light-manager contract closely enough to compare against RTX Portal
and RTX Remix debugging behavior.


Contract C-01: Current Light Producer Input
-------------------------------------------

rbdoom producers:

    PathTraceDoomLights.cpp
    PathTraceSmokeSceneBuild.cpp
    PathTraceLightUniverse.cpp
    PathTraceUnifiedLight.cpp

Required input:

    current Doom analytic light records
    current emissive light records, after analytic proof
    stable source identity
    light type
    payload fields needed by RAB
    current source-domain count

Pass:

    source inventory is broad enough for real lighting
    proposal count and RTXDI-facing dense count are not confused
    every emitted current light has a replayable RAB payload

Fail:

    authored universe is shrunk to hide noise
    old NEE records are used as light identity
    portal/mirror/subview accidents are promoted as the active solution


Contract C-02: Dense Current And Previous Domains
-------------------------------------------------

Reference shape:

    RTX Remix LightManager::prepareSceneData
    RTX Remix common_bindings.slangh lights / previousLights

Required output:

    dense current light payload array
    dense previous light payload array
    current active light count
    previous active light count
    typed ranges over the current dense array

Pass:

    current reservoir light indices index the current dense array
    previous reservoir light indices index the previous dense array
    disabled/off lights do not leave live dense slots unless explicitly encoded
        as valid zero-payload diagnostics

Fail:

    shader-facing previous payloads are sized from a stale candidate slot
        concept
    persistent authored registry order is treated as the RTXDI light index
    missing/off/diagnostic records alter unrelated dense indices


Contract C-03: Two-Way Light Mapping
------------------------------------

Reference shape:

    RTX Remix RAB_TranslateLightIndex
    RTXDI DI TemporalResampling.hlsli

Required output:

    current-to-previous map for bias correction
    previous-to-current map for previous reservoir reuse
    explicit invalid sentinel for unmapped lights

Pass:

    a current dense index maps to its previous dense index when the same light
        exists in both domains
    a previous dense index maps to its current dense index when the same light
        exists in both domains
    a failed mapping rejects only that previous/current light candidate

Fail:

    mapping validity is decided by broad authored-registry diagnostics instead
        of actual current/previous dense-domain membership
    one animated or temporary light invalidates unrelated stable lights
    mapping requires full-map brute force or broad suppression


Contract C-04: Animated Payloads
--------------------------------

Reference shape:

    Remix keeps buffer identity while payload changes.

Required behavior:

    color/intensity/radius/position changes are payload changes
    payload changes do not invalidate identity by themselves
    payload-only changes may update payload signatures and diagnostics

Pass:

    animated room lights keep mapping when their identity remains stable
    material-register animation can be replaced by game-authored color for
        diagnostics without changing mapping semantics

Fail:

    every radiance change invalidates remap
    animated payload churn is treated as structural identity failure
    animation from one light leaks into other light payloads


Contract C-04A: Reusable Analytic Stability Classification
---------------------------------------------------------

Required output:

    currentSampleable flag for same-frame direct-light replay
    stableCacheable flag for persistent cache/ReGIR/temporal-style reuse
    stable Doom analytic prefix/count inside the Doom analytic typed range
    unstableDynamic flag for current-frame-only analytics
    explicit rejection reason flags:
        noRemap
        payloadChanged
        unprovenContinuity
        unknownIdentity
        duplicateIdentity
        portalDisconnected
        outOfSelectedArea

Pass:

    stableCacheable requires stable identity, valid current-to-previous and
        previous-to-current remaps, no duplicate identity, proven continuity,
        and no incompatible payload change
    if any required proof is missing, the analytic remains currentSampleable
        only and is marked unstableDynamic with a reason
    portal depth metadata alone is not outOfSelectedArea proof and must not
        reject otherwise stable analytic lights from stableCacheable
    consumers keep dense RLU identity and replay through RAB only
    persistent consumers sample only the stableCacheable prefix/count, while
        current-frame direct lighting may still use the full current range
    fallback-only current-frame proposals may use currentSampleable lights but
        must not store unstableDynamic analytics into persistent cells

Fail:

    persistent consumers cache every current Doom analytic in the typed range
    consumers dereference DoomAnalyticLights[sourceIndex] or run private
        continuity tests
    animated current-frame lights poison reusable cell or temporal state


Contract C-05: Spawn / Despawn / Effect Lights
----------------------------------------------

Reference shape:

    Remix maps unmatched new lights to invalid until matched; disappearance is
    local.

Required behavior:

    spawned lights can enter the current dense array
    despawned lights can remain only in previous dense array for translation
        rejection
    unmatched lights map invalid without poisoning unrelated lights

Pass:

    sparks, robot lights, and effect lights cause only local current-only noise
    stable room lights keep valid mappings while effects spawn/despawn

Fail:

    temporary effects change stable room-light remaps
    unmapped current-only lights reset the whole temporal domain
    broad effect suppression is required for stable room-light accumulation


Contract C-06: RAB Bridge
-------------------------

Required shader behavior:

    RAB_LoadLightInfo(index, false) loads current dense payload
    RAB_LoadLightInfo(index, true) loads previous dense payload
    RAB_TranslateLightIndex(index, true) maps current to previous
    RAB_TranslateLightIndex(index, false) maps previous to current

Pass:

    RTXDI temporal can translate and replay lights without knowing rbdoom's
        source registry internals
    failed translation rejects only the previous sample

Fail:

    RAB reloads legacy split payload arrays through unstable compact indices
    shader code has to inspect authored-registry invalid reasons to translate
    final resolve uses a different light domain than temporal


Contract C-07: Reset And Lifetime
---------------------------------

Required behavior:

    map change clears current/previous/mapping state
    render-world change clears current/previous/mapping state
    buffer resize does not expose stale payloads as valid
    first frame after reset is explicitly current-only

Pass:

    one-shot dumps report clear reason and dense counts
    stale previous-map lights cannot be translated

Fail:

    old map lights survive as valid previous payloads
    history resets hide a broken remap contract


Contract C-08: Diagnostics Versus Proof
---------------------------------------

Required diagnostics:

    dense current count
    dense previous count
    mapped current-to-previous count
    mapped previous-to-current count
    current-only count
    previous-only count
    invalid duplicate identity count
    payload-only signature change
    structural signature change

Pass:

    dumps explain why a light maps or does not map
    visual proof still comes from raw temporal accumulation

Fail:

    green status bands, CPU dumps, or allocation success are reported as
        accumulation proof


Contract C-09: Active Polymorphic Buffer Ownership
--------------------------------------------------

Required shader-visible ownership:

    register(t66) is the active current RLU polymorphic payload buffer
    register(t67) is the active previous RLU polymorphic payload buffer
    PathTraceUnifiedLights at t59/t60 are legacy/debug unless explicitly named
        by a task
    selected light identity is the dense current RLU index

Pass:

    clean DI, PDFNEE, NEE-cache, and ReGIR consumers load active lights through
        RAB_LoadActiveRrxLightInfo or the equivalent active RAB load helper
    light sampling uses RAB_SampleActiveRrxPolymorphicLight or
        RAB_SamplePolymorphicLight over RAB-loaded dense RLU records
    new modules consume the RLU rather than re-ingesting Doom analytics or
        emissives privately

Fail:

    a consumer builds its own analytic/emissive light universe
    a consumer uses sourceIndex to reload old Doom analytic arrays as the
        normal path
    NEE cache, ReGIR cells, portal pools, or split-domain arrays become the
        default clean DI light universe


Contract C-10: Authoritative Typed Ranges And Sample Counts
-----------------------------------------------------------

Required output:

    RestirLightManagerRangeInfo.xy = emissive dense offset/count
    RestirLightManagerRangeInfo.zw = Doom analytic dense offset/count
    RestirLightManagerSampleInfo.x = bounded emissive initial sample count
    RestirLightManagerSampleInfo.y = bounded Doom analytic initial sample count
    RestirLightManagerSampleInfo.z = total requested proposal count
    RestirLightManagerSampleInfo.w = non-empty typed range count

Pass:

    shader consumers can compute proposal PDFs from range count, range sample
        count, and selected source policy
    zero-count ranges are valid and do not trigger private fallback domains
    dumps and shader helpers report the same range/sample contract

Fail:

    normal shader operation scans the payload buffer to rediscover ranges
    range sample count and light count are conflated
    typed range metadata is owned by NEE-cache or ReGIR instead of RLU


Contract C-11: Clean DI Proposal Sampling From RLU
--------------------------------------------------

Required behavior:

    clean RTXDI DI initial sampling takes proposal candidates from active RLU
        typed ranges
    sourcePdf is the actual proposal probability for the selected dense RLU
        light
    targetPdf comes from RAB_GetLightSampleTargetPdfForSurface
    solidAnglePdf comes from RAB polymorphic light sampling
    RTXDI helper reservoir math finalizes the current reservoir with M = 1

Pass:

    changing bounded sample counts predictably changes noise while preserving
        mean lighting
    emissive and analytic candidates can both enter the current reservoir when
        their typed ranges are non-empty
    current reservoir identity remains dense current RLU index

Fail:

    visibility is folded into targetPdf
    final output clamping is used as raw current-frame proof
    NEE-cache or ReGIR is required for normal clean DI function
    sourcePdf is uniform when typed/range sampling selected the light through a
        different proposal distribution


Contract C-12: Previous-Best Initial-Sampling Seed
--------------------------------------------------

Required behavior:

    previous-best feedback is applied during initial sampling, before temporal
        reuse
    previous-domain light index is translated with
        RAB_TranslateLightIndex(previousLight, false)
    the translated current-domain light is sampled and combined with random
        candidates through RTXDI reservoir helpers
    failed translation rejects only that previous-best candidate

Pass:

    previous-best seeding improves movement stability in overlapping-light
        areas without changing stationary lighting mean
    any rbdoom approximation that uses a previous temporal reservoir selected
        light is labeled as an approximation, not full Remix best-light parity

Fail:

    temporal reuse is rewritten to compensate for bad current proposals
    previous best skips RAB translation
    previous-domain indices are used directly as current indices
    compliance is claimed without a real best-light feedback source or without
        explicitly documenting the approximation
