PDF + NEE Replacement Worker Tasks
==================================

Run one task at a time. Stop after the first failing task or after build/deploy
for user visual testing.


Current Status
--------------

The old PDFNEE/ReGIR verifier implementation is discarded as an implementation
base. It failed the basic proof requirement and was built before the current
working RLU path existed.

Do not launch or extend old PDFNEE-00 through PDFNEE-11 tasks. Do not implement
from their light modes, handoffs, candidate pools, ReGIR recipes, or old NEE
records.

The current known-good chain is:

    Remix Light Universe (RLU) dense current/previous domain
    active RRX RAB helpers over that dense RLU domain
    clean RTXDI DI view 12 consuming RLU
    clean RTXDI DI temporal accumulation as the proof consumer

Any replacement PDF/NEE module must be built from this chain:

    one enable CVar = normal current-frame DI/NEE functionality
    default light source = current RLU dense domain
    default proposal PDF = derived from the current accepted RLU source policy
    default handoff target = clean RTXDI DI current reservoir page
    extra CVars = diagnostics only

Current checked-out state:

    PDFNEE-RLU-02 full dense uniform remains the baseline sourcePolicy 0
    PDFNEE-RLU-03 fixed 50/50 typed policy is superseded
    PDFNEE-RLU-04 is the active sourcePolicy 1 range-stratified producer
    current blocker is PDFNEE analytic-light parity with clean RTXDI DI view 12

Do not spend more worker time on temporal accumulation, motion vectors, history
length, depth thresholds, denoising, or fallback lighting until PDFNEE current
Doom analytic lighting matches clean RTXDI DI view 12 for the same RLU domain.

Domain alignment rule:

    RLU-05 comparisons must explicitly prove both PDFNEE/RLU and clean RTXDI DI
    view 12 are using the same RLU domain. With only
    r_pathTracingRestirPdfNeeVerifierEnable 1, scene build requests RLU domain 2
    when r_pathTracingRemixLightUniverseEnable is off, so the one-CVar route
    uses the mixed emissive + Doom analytic dense current domain by default.
    If r_pathTracingRemixLightUniverseEnable is on, the explicit
    r_pathTracingRemixLightUniverseDomain value is authoritative. Domain 0
    analytic-only results still must not be compared against domain 2 mixed-RLU
    clean view 12 results.

Historical stale code text:

    Older r_pathTracingRestirPdfNeeVerifierSourcePolicy CVar help said policy 1
    was typed 50/50. Treat that wording as stale wherever it appears. The
    current shader/docs contract is RLU-04 range-stratified:
        sourcePdf = rangeSampleCount/(rangeCount*totalProposalSamples)
        invSourcePdf = (rangeCount*totalProposalSamples)/rangeSampleCount

PDFNEE-RLU-01: Default Current-Frame Producer Design
----------------------------------------------------

Goal:

    Design, but do not yet implement, the replacement current-frame producer
    that uses the current RLU dense domain by default.

Allowed write set:

    docs/restir_pdf_nee_verifier only

Output artifact:

    current_frame_producer_design.txt

Required design:

    one enable CVar activates the default producer
    default source domain is the RLU dense current domain
    default output writes a clean RTXDI DI compatible current reservoir page or
        raw current-frame direct-light output
    default route does not require a view/mode/debug recipe
    diagnostics are optional and cannot be required for functionality
    selected reservoir identity is a dense RLU current-domain index
    source policy is full-domain uniform over dense RLU current lightIndex
        range [0, currentRluLightCount)
    sourcePdf = 1/currentRluLightCount and
        invSourcePdf = currentRluLightCount for every first-producer proposal
    RTXDI helper proposal count M is the number of streamed candidates, not a
        substitute for the source domain size
    solidAnglePdf, targetPdf, reflected radiance, visibility, and final
        contribution all use the same replayed selected sample

Forbidden:

    no payload sourcePdf/sourceWeight sampling in the first producer
    no typed-range uniform or bounded per-range sampling in the first producer
    no old PDFNEE lightMode 8 or 9 route
    no ReGIR dependency
    no Doom portal candidate pool as the default producer
    no split-domain fallback as the default producer
    no old rbdoom selected-light NEE records
    no old rbdoom NEE contribution output
    no PathTraceSmokeReservoir handoff
    no mode 56, view 72, RRX pages, spatial reuse, best-light, or denoiser path
    no broad light suppression
    no temporal accumulator edits

Required source policy:

    Use the current RLU typed ranges:

        emissive range:
            [emissiveOffset, emissiveOffset + emissiveCount)

        Doom analytic range:
            [doomAnalyticOffset, doomAnalyticOffset + doomAnalyticCount)

    For each streamed candidate:

        choose one non-empty class with an explicitly stated classProbability
        choose lightIndex uniformly inside that class
        sourcePdf = classProbability / classCount
        invSourcePdf = classCount / classProbability
        selected reservoir identity remains dense current RLU lightIndex

    The sum of classProbability over non-empty classes must equal 1.0.

        if both emissive and Doom analytic ranges are non-empty:
            classProbability(emissive) = 0.5
            classProbability(doomAnalytic) = 0.5

            classProbability(the class) = 1.0

    ReGIR cell weights, light power, or alias tables in this task.

        and final contribution shape unchanged except for sourcePdf/invSourcePdf.
    Keep the one-CVar normal route usable.
    If a source-policy CVar is added, the proven typed policy may become the
        default only after the full-domain uniform policy remains available as a
        diagnostic comparison.

Forbidden:

    no temporal accumulator edits
    no spatial reuse
    no best-light, denoiser, mode 56, RRX pages, or ReGIR dependency
    no portal candidate pool
    no payload sourcePdf/sourceWeight sampling
    no broad light suppression

PDFNEE-RLU-04: Range-Stratified RLU Current Producer
----------------------------------------------------

Status:

    Active checked-out implementation contract for
    r_pathTracingRestirPdfNeeVerifierSourcePolicy 1.

Goal:

    Keep the one-CVar PDFNEE/RLU current producer on the current RLU typed
    ranges while using an explicitly normalized range-stratified proposal PDF.

Required rbdoom read set:

    neo/shaders/builtin/pathtracing/pathtrace_smoke.rt.hlsl
    neo/shaders/builtin/pathtracing/pathtrace_restir_pdf_nee_rlu_current.rt.hlsl
    neo/shaders/builtin/pathtracing/RtxdiBridge/RAB_LightLoadRuntime.hlsli
    neo/shaders/builtin/pathtracing/RtxdiBridge/RAB_LightSamplingCore.hlsli
    neo/shaders/builtin/pathtracing/RtxdiBridge/RAB_LightTarget.hlsli
    neo/renderer/NVRHI/PathTraceSmokeDispatch.cpp
    neo/renderer/NVRHI/PathTraceSmokeSceneBuild.cpp

Required source policy:

    Use raw current RLU typed ranges:

        emissive range:
            [emissiveOffset, emissiveOffset + emissiveCount)

        Doom analytic range:
            [doomAnalyticOffset, doomAnalyticOffset + doomAnalyticCount)

    For every streamed candidate in a range:

        lightIndex = dense current RLU lightIndex inside that range
        rangeSampleCount = streamed proposal count for that range
        totalProposalSamples = sum of streamed range proposal counts
        sourcePdf = rangeSampleCount / (rangeCount * totalProposalSamples)
        invSourcePdf = (rangeCount * totalProposalSamples) / rangeSampleCount
        selected identity = dense current RLU lightIndex

    Do not call this fixed 50/50. Do not infer sourcePdf from payload
    sourcePdf/sourceWeight, light power, ReGIR cells, portal data, or old NEE
    records.

Required unchanged contracts:

    RTXDI_StreamSample
    RTXDI_FinalizeResampling with streamed proposal count
    stored current reservoir M = 1
    selected sample UV replay
    targetPdf from RAB_GetLightSampleTargetPdfForSurface
    selected visibility stored separately
    final contribution shape
    clean current reservoir page u69

Acceptance:

    dump reports sourcePolicy 1 as range-stratified, not fixed 50/50
    dump reports sourcePdfFormula =
        rangeSampleCount/(rangeCount*totalProposalSamples)
    dump reports invSourcePdfFormula =
        (rangeCount*totalProposalSamples)/rangeSampleCount
    clean RTXDI DI can consume the produced current page
    temporal-off current output keeps plausible mean relative to sourcePolicy 0
    user visual testing confirms no obvious mean-brightness drift
