PDFNEE-01: Verifier CVar And Route Skeleton
-------------------------------------------

Goal:

    Add an isolated verifier route that owns every pixel and cannot use
    temporal/spatial/denoiser/fallback output.

Reference file-equivalent:

    E:/prog/references/dxvk-remix-git/src/dxvk/shaders/rtx/pass/rtxdi/rtxdi_initial_sampling.comp.slang

Allowed rbdoom write set:

    verifier CVar definitions
    standalone verifier shader or mode
    minimal dispatch/resource binding for verifier output
    CMake shader registration if needed

Forbidden:

    no temporal
    no spatial
    no best lights
    no existing clean synthetic accumulator edits
    no mode-56 route

Acceptance:

    route sentinel visible
    invalid/no-light output is loud
    verifier does not expose fallback scene lighting


PDFNEE-02: One Synthetic Light Estimator Proof
----------------------------------------------

Goal:

    Prove the final contribution formula with one known light.

Required:

    source PDF = 1.0
    positive solidAnglePdf
    positive targetPdf
    raw flat-diffuse current-frame output
    no temporal

Acceptance:

    output matches documented expected contribution
    estimator table is included in report


PDFNEE-03: Synthetic Overlap Estimator Proof
--------------------------------------------

Goal:

    Prove source-PDF normalization and contribution under two and N overlapping
    lights.

Required:

    two-light source PDF table
    N-light source PDF table
    source PDF sums to 1.0
    selected light identity changes do not alter mean brightness incorrectly

Acceptance:

    noisy one-sample output has correct mean
    no strobe from light identity weighting errors
