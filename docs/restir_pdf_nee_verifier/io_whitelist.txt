PDF + NEE Replacement I/O Whitelist
===================================

This whitelist applies to replacement current-frame PDF/NEE work.

The active source of truth is the Remix Light Universe (RLU) dense current
domain. The active handoff target is the clean RTXDI DI current reservoir page.

Old split-domain, portal-pool, ReGIR, and rbdoom NEE inputs are discarded as
implementation routes. They may not be used as default producer sources or
proof outputs.


Allowed Inputs
--------------

I-01: Current Primary Surface

    Meaning:
        Current-frame surface data needed to evaluate direct lighting.

    Allowed fields:
        world position
        geometric normal
        shading normal
        view direction
        material/roughness/specular data if using shared RAB BRDF
        valid flag

    Forbidden use:
        previous surface
        motion vectors
        temporal reprojection


I-02: Current RLU Dense Light Domain

    Meaning:
        The current dense RLU light domain used by the replacement producer.

    Allowed sources:
        current RLU dense light payload records
        current RLU typed ranges
        current RLU sample-count metadata

    Required declaration:
        worker must name exactly which RLU domain is active and the current
        dense count used for proposal selection.

    Required first producer source policy:
        proposal domain is the full current RLU dense lightIndex range
            [0, currentRluLightCount)
        sourcePdf = 1.0 / currentRluLightCount
        invSourcePdf = currentRluLightCount
        selected reservoir identity is the dense current lightIndex

    Current accepted alternate source policy:
        PDFNEE-RLU-04 sourcePolicy 1 may sample raw current RLU typed ranges
        with an explicitly normalized rangeSampleCount/rangeCount source PDF.
        This is allowed only because sourcePolicy 0 remains available as the
        full dense uniform baseline.

    Forbidden use:
        RLU payload sourcePdf/sourceWeight as the first producer proposal PDF
        Doom analytic payload sourcePdf=0 as a valid proposal PDF
        fixed 50/50 typed-range sampling as the current sourcePolicy 1 contract
        unapproved bounded per-range sampling as the first producer source
            policy
        payload-weighted or alias-table sampling as the first producer source
            policy
        split current emissive + Doom analytic arrays as the default source
        unified current light records as the default source
        legacy ReSTIR light-manager records as the default source
        ReGIR candidate cache as the default source
        portal-prioritized candidate pool as the default source
        previous light domain during current-frame proof
        broad pruning to avoid overlap failures


I-03: Active RLU RAB Helpers

    Required helpers:
        RAB_LoadActiveRrxLightInfo
        RAB_SampleActiveRrxPolymorphicLight
        RAB_GetLightSampleTargetPdfForSurface
        RAB_GetReflectedBsdfRadianceForSurface

    Allowed use:
        default replacement producer implementation
        clean RTXDI DI current reservoir producer

    Forbidden use:
        replacing these helpers with local light math
        falling back to split-domain RAB helpers without a loud diagnostic and
            manager approval
        locally duplicated helper implementations unless exact equivalence is
            proven before runtime proof


I-04: Light Identity Metadata

    Meaning:
        Light identity/type/source data used to classify selected samples and
        validate replay.

    Allowed use:
        current-frame diagnostic labels
        selected-light identity table

    Required identity:
        selected reservoir identity is the dense RLU current-domain index that
        the clean RLU consumer can replay.

    Forbidden use:
        old rbdoom selected-light NEE identity
        compact portal candidate index
        ReGIR candidate slot
        previous-frame remap as proof of current-frame PDF correctness


I-04B: Current Doom Analytic Payload Parity

    Meaning:
        Current-domain Doom analytic RLU records may carry a sourceIndex that
        points at the uploaded DoomAnalyticLights payload used by clean RTXDI DI
        view 12.

    Allowed use:
        PDFNEE/RLU current producer may match clean view-12 current analytic
        loading by resolving the dense RLU record sourceIndex through
        DoomAnalyticLights[sourceIndex], then preserving the dense RLU lightIndex
        as the selected reservoir identity.

    Required declaration:
        worker must state whether a selected Doom analytic light was loaded
        through the clean-compatible direct DoomAnalyticLights payload or
        through a packed unified RLU record.

    Forbidden use:
        changing selected reservoir identity from dense current RLU lightIndex
        using legacy PDFNEE analytic view output as the baseline
        changing temporal accumulation to hide current analytic load mismatch


I-05: Visibility Helper

    Allowed use:
        forced-visible one-light calibration
        traced visibility real-light validation

    Required declaration:
        worker must state whether a rung uses forced visibility or traced
        visibility.

    Forbidden use:
        fallback lighting when visibility rejects


I-06: RTXDI DI Reservoir Helpers

    Allowed helpers:
        RTXDI_EmptyDIReservoir
        RTXDI_StreamSample
        RTXDI_FinalizeResampling
        RTXDI_PackDIReservoir
        RTXDI_GetDIReservoirInvPdf
        RTXDI_GetDIReservoirLightIndex
        RTXDI_GetDIReservoirSampleUV

    Allowed use:
        current-frame initial reservoir only

    Forbidden use:
        temporal reuse in current-frame proof
        spatial reuse
        previous-best feedback
        hand-authored packed reservoir fields without exact helper-equivalence
            proof


Forbidden Inputs
----------------

Do not consume:

    previous reservoir pages for current-frame proof
    temporal reservoir pages for current-frame proof
    spatial reservoir pages
    previous primary surface
    motion vector textures
    mode-56 RRX debug output
    view-72 output
    old rbdoom selected-light NEE records
    old rbdoom NEE contribution output
    split-domain Doom analytic arrays as the default source
    ReGIR candidate cache as the default source
    portal-prioritized candidate pool as the default source
    denoiser/confidence/gradient buffers
    game beauty/albedo fallback output
    synthetic temporal accumulator history
    compact portal proposal index as final reservoir light identity
    ReGIR debug color output as a light sample
    ReGIR temporal/spatial/best-light state


Allowed Outputs
---------------

O-01: Raw Flat-Diffuse Direct-Light View

    Required:
        physically based direct-light estimate
        constant diffuse reflectance
        current-frame only
        one sampled candidate per pixel for first proof
        no temporal average
        no albedo fallback
        no old NEE contribution output


O-02: Clean RTXDI DI Current Reservoir Page

    Required:
        RTXDI_PackedDIReservoir-compatible current page
        selected light identity in the clean RLU consumer domain
        sample UV matching the RAB sample and targetPdf evaluation
        sourcePdf and targetPdf produced by the same selected sample contract
        invalid paths write empty reservoirs

    Allowed use:
        default replacement producer output
        clean RTXDI DI consume proof


O-03: Numeric Probe View

    Allowed fields:
        lightSelectionPdf or invSourcePdf
        solidAnglePdf
        targetPdf
        reservoir inverse PDF / weight
        reflectedRadiance magnitude
        finalContribution magnitude

    Restriction:
        diagnostic only; cannot be success proof without O-01 or O-02.


O-04: Loud Invalid Status View

    Required invalid states:
        invalid surface
        no RLU light domain
        invalid light info
        invalid replay sample
        zero source PDF
        zero solidAnglePdf
        zero targetPdf
        visibility rejected

    Restriction:
        diagnostic only.


Forbidden Outputs
-----------------

Do not output:

    temporal accumulation proof before clean consume
    spatial proof
    best-light proof
    denoised output
    game beauty output
    albedo fallback
    status colors as final estimator proof
    old rbdoom NEE selected-light records
    old rbdoom NEE contribution output


Reject Condition
----------------

Reject any worker result that cannot list every consumed input and written
output by whitelist id.
