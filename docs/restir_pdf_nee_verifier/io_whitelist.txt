PDF + NEE Verifier I/O Whitelist
================================

This whitelist applies only to the standalone current-frame verifier.


Allowed Inputs
--------------

I-01: Current Primary Surface

    Meaning:
        Current-frame surface data needed to evaluate direct lighting.

    Allowed fields:
        world position
        geometric normal
        shading normal
        view direction
        material/roughness/specular data if using shared RAB BRDF
        valid flag

    Forbidden use:
        previous surface
        motion vectors
        temporal reprojection


I-02: Current Light Domain

    Meaning:
        The current rbdoom light universe / RAB light domain used by the
        verifier.

    Allowed sources:
        split current emissive + Doom analytic arrays
        unified current light records
        ReSTIR light-manager current payload records

    Required declaration:
        worker must name exactly which domain is active.

    Forbidden use:
        previous light domain during verifier proof
        broad pruning to avoid overlap failures


I-03: Light Identity Metadata

    Meaning:
        Light identity/type/source data used to classify selected samples and
        validate replay.

    Allowed use:
        current-frame diagnostic labels
        selected-light identity table

    Forbidden use:
        previous-frame remap as proof of current-frame PDF correctness


I-04: Shared RAB Light Helpers

    Required helpers:
        RAB_LoadLightInfo
        RAB_SamplePolymorphicLight
        RAB_GetLightSampleTargetPdfForSurface
        RAB_GetReflectedBsdfRadianceForSurface

    Allowed use:
        all verifier estimator rungs

    Forbidden use:
        locally duplicated helper implementations unless exact equivalence is
        proven before runtime proof


I-05: Visibility Helper

    Allowed use:
        forced-visible one-light calibration
        traced visibility real-light validation

    Required declaration:
        worker must state whether a rung uses forced visibility or traced
        visibility.

    Forbidden use:
        fallback lighting when visibility rejects


I-06: RTXDI DI Reservoir Helpers

    Allowed helpers:
        RTXDI_StreamSample
        RTXDI_SampleLocalLights
        RTXDI_FinalizeResampling
        RTXDI_GetDIReservoirInvPdf
        RTXDI_GetDIReservoirLightIndex
        RTXDI_GetDIReservoirSampleUV

    Allowed use:
        current-frame initial reservoir only

    Forbidden use:
        temporal reuse
        spatial reuse
        previous-best feedback


Forbidden Inputs
----------------

Do not consume:

    previous reservoir pages
    temporal reservoir pages
    spatial reservoir pages
    previous primary surface
    motion vector textures
    mode-56 RRX debug output
    old rbdoom selected-light NEE records
    old rbdoom NEE contribution output
    denoiser/confidence/gradient buffers
    game beauty/albedo fallback output
    synthetic temporal accumulator history


Allowed Outputs
---------------

O-01: Raw Flat-Diffuse Direct-Light View

    Required:
        physically based direct-light estimate
        constant diffuse reflectance
        current-frame only
        one sampled candidate per pixel for first proof
        no temporal average
        no albedo fallback


O-02: Numeric Probe View

    Allowed fields:
        lightSelectionPdf
        solidAnglePdf
        targetPdf
        reservoir inverse PDF / weight
        reflectedRadiance magnitude
        finalContribution magnitude

    Restriction:
        diagnostic only; cannot be success proof without O-01.


O-03: Loud Invalid Status View

    Required invalid states:
        invalid surface
        no light domain
        invalid light info
        invalid replay sample
        zero source PDF
        zero solidAnglePdf
        zero targetPdf
        visibility rejected

    Restriction:
        diagnostic only.


Forbidden Outputs
-----------------

Do not output:

    temporal accumulation proof
    spatial proof
    best-light proof
    denoised output
    game beauty output
    albedo fallback
    status colors as final estimator proof


Reject Condition
----------------

Reject any worker result that cannot list every consumed input and written
output by whitelist id.
