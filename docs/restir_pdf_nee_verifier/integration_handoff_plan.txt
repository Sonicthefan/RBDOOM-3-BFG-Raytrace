RLU Replacement Producer Handoff Plan
=====================================

Scope
-----

This file supersedes the old PDFNEE-08/PDFNEE-09/PDFNEE-10/PDFNEE-11 handoff
plan.

The old PDFNEE/NEE attempts were built before the current working Remix Light
Universe (RLU) path existed. They are discarded as implementation routes. Do
not retain, extend, or reactivate them. They may be read only as failure
evidence for what not to use.


Working Baseline
----------------

Known-good producer/consumer chain:

    RLU dense current/previous light domain
    active RRX RAB load/sample helpers over that dense domain
    clean RTXDI DI current reservoir contract
    clean RTXDI DI view 12 consuming RLU on real Doom lighting

Do not modify the clean RTXDI DI temporal accumulator for the replacement
producer task.


Replacement Target
------------------

The replacement current-frame producer must target this contract:

    current RLU dense light index
    RAB_LoadActiveRrxLightInfo(...)
    RAB_SampleActiveRrxPolymorphicLight(...)
    RAB_GetLightSampleTargetPdfForSurface(...)
    RAB_GetReflectedBsdfRadianceForSurface(...)
    RTXDI current reservoir page equivalent to CleanRtxdiDiCurrentReservoirs
        register(u69)
    clean RTXDI DI consumer replaying the same dense RLU identity and sample UV

The selected reservoir light identity is the dense RLU current-domain index
consumed by the clean RLU path. It is not an old split analytic index, compact
portal candidate index, ReGIR candidate slot, or old NEE record identity.


Initial Source PDF Policy
-------------------------

The first replacement producer must match the working clean path's effective
initial-sampling policy: full-domain uniform sampling over the current RLU
dense domain.

For currentRluLightCount > 0:

    proposal domain = dense current lightIndex range [0, currentRluLightCount)
    sourcePdf = 1.0 / currentRluLightCount
    invSourcePdf = currentRluLightCount
    selected reservoir light identity = dense current lightIndex

The RLU payload sourcePdf/sourceWeight fields are not the first producer's
proposal PDF. Doom analytic payload sourcePdf values may be zero, so using
payload sourcePdf would create an invalid first proof.

Typed-range uniform, bounded per-range, payload-weighted, alias-table, portal,
and ReGIR sampling are deferred policies. Each requires a separate task with a
new source PDF proof before implementation.

The RTXDI helper proposal count M is the number of candidate samples streamed
into the reservoir. Do not replace the source PDF domain size with M.

If currentRluLightCount is zero, write an empty reservoir or loud invalid
diagnostic. Do not fall back to an old light domain.


Required Producer Data
----------------------

For each current-frame candidate:

    sourcePdf or invSourcePdf for the full current RLU dense proposal domain
    sample UV used for RAB replay
    RAB_LightInfo from RAB_LoadActiveRrxLightInfo
    RAB_LightSample from RAB_SampleActiveRrxPolymorphicLight
    solidAnglePdf from that same sample
    targetPdf from RAB_GetLightSampleTargetPdfForSurface
    reflected radiance from RAB_GetReflectedBsdfRadianceForSurface
    visibility policy, explicit per proof rung
    RTXDI reservoir weight/finalization through RTXDI helpers

Invalid candidates must write empty current reservoirs or loud diagnostics.
They must not preserve previous reservoirs, reveal beauty/albedo fallback, or
fall through to old NEE output.


Default Functionality
---------------------

One replacement enable CVar must activate the normal current-frame producer
with useful defaults:

    default source = current RLU dense domain
    default output = useful raw current-frame lighting or clean-compatible
        current reservoir page
    diagnostics = optional, not required for normal function

A long recipe involving PDFNEE light modes, ReGIR, portal pools, mode 56,
view 72, split-domain switches, or clean-side debug toggles is rejected.


Discarded Routes
----------------

Do not implement from these old routes:

    old PDFNEE lightMode 8 RLU diagnostic
    old PDFNEE lightMode 9 ReGIR consume
    old PDFNEE-09 external consume handoff
    old PDFNEE-10 portal-prioritized analytic candidate pool
    old PDFNEE-11 ReGIR source distribution handoff
    PathTraceSmokeReservoir handoff
    old rbdoom selected-light NEE records
    old rbdoom NEE contribution output
    ReGIR debug views
    mode 56 or view 72

Existing code for those paths is legacy/failure evidence only unless a later
manager task explicitly asks to delete it. It must not be used as proof or as
the implementation base for the replacement producer.


First Integration Proof
-----------------------

After the current-frame producer is implemented and built:

    temporal off:
        current output or current page has the expected noisy mean
        no fallback beauty/albedo
        no old NEE record path

    clean consume:
        clean RTXDI DI consumes the replacement current page
        selected light identity replays through the dense RLU RAB path
        sample UV, solidAnglePdf, targetPdf, and reservoir weight are from the
            same selected sample contract

    temporal on, only after current-frame proof:
        clean RTXDI DI reduces noise without shifting mean brightness

Stop if sourcePdf/invSourcePdf, sample UV, targetPdf, solidAnglePdf, selected
identity, or RTXDI helper sequence cannot be stated exactly.
