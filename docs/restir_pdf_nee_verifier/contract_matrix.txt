PDF + NEE Replacement Contract Matrix
=====================================

Goal
----

Verify that rbdoom can produce a correct current-frame direct-light reservoir
estimate from the current Remix Light Universe (RLU) domain and hand it to the
known-good clean RTXDI DI consumer.

The old split-domain PDFNEE/ReGIR/portal/NEE attempts are discarded as
implementation routes. They were built before the current working RLU path
existed. Do not retain, extend, or reactivate them for this replacement.


Contract C-00: Default RLU Producer Shape
-----------------------------------------

Producer:

    PathTraceRemixLightManager.cpp
    PathTraceSmokeSceneBuild.cpp
    PathTraceSmokeDispatch.cpp

Required source:

    current RLU dense payload domain
    current RLU typed ranges
    active RRX RAB light load/sample helpers

Required default behavior:

    one enable CVar activates useful current-frame DI/NEE functionality
    default output is raw current-frame lighting or a clean-compatible current
        reservoir page
    extra CVars are diagnostic only
    first replacement source policy is full-domain uniform over the current
        RLU dense domain

Pass:

    no debug view, ReGIR mode, portal mode, split fallback, mode 56, view 72,
        or old NEE record path is required for normal functionality

Fail:

    normal behavior requires a long CVar recipe
    output is only status colors, grey, black, or fallback beauty
    default source is ReGIR, portal candidates, split Doom arrays, or old NEE
        records
    broad light suppression is used as a fix


Contract C-01: CPU Light Payload Domain
---------------------------------------

Producer:

    PathTraceRemixLightManager.cpp
    PathTraceSmokeSceneBuild.cpp

Required output:

    current RLU dense light records
    light type
    position/radius
    area
    radiance/luminance
    identity/remap metadata
    typed ranges
    current-to-previous and previous-to-current maps for later clean temporal
        consume only
    current dense count used by the first producer as the full proposal
        domain size

Not owned here:

    solidAnglePdf
    targetPdf
    final NEE/direct-light contribution

Pass:

    worker proves the RLU dense current count, typed ranges, and payload buffer
    worker states how the replacement producer selects from the RLU domain

Fail:

    worker assumes old split/unified managers are the default producer
    worker shrinks authored light universe as a fix


Contract C-02: Light Load Adapter
---------------------------------

Reference rbdoom file:

    neo/shaders/builtin/pathtracing/RtxdiBridge/RAB_LightLoadRuntime.hlsli

Required call:

    RAB_LoadActiveRrxLightInfo(lightIndex, false)

Pass:

    every sampled dense RLU lightIndex resolves to valid RAB_LightInfo or a
        loud invalid state
    current domain count matches the proposal domain

Fail:

    verifier defines a duplicate RAB load helper without proving exact
        equivalence
    previous-frame load is used in current-frame proof
    old split-domain or old NEE identity is used as the normal identity


Contract C-03: Light Sampling And Replay
----------------------------------------

Reference rbdoom file:

    neo/shaders/builtin/pathtracing/RtxdiBridge/RAB_LightSamplingCore.hlsli

Required call:

    RAB_SampleActiveRrxPolymorphicLight(lightInfo, surface, uv)

Required output:

    valid flag
    sample position
    light direction/distance
    radiance
    solidAnglePdf

Pass:

    selected sample can be replayed from dense RLU reservoir lightIndex + uv
    solidAnglePdf is positive for valid selected samples
    invalid replay is loud

Fail:

    selected sample position is reconstructed differently in resolve than in
        targetPdf
    solidAnglePdf is omitted or replaced by a debug scalar


Contract C-04: Source PDF
-------------------------

Reference math:

    RTXDI DI InitialSampling.hlsli
    RTXDI DI Reservoir.hlsli

Required proof:

    first replacement producer uses the full current RLU dense domain:
        lightIndex in [0, currentRluLightCount)
    selected reservoir light identity is that dense current-domain index
    sourcePdf = 1.0 / currentRluLightCount for every candidate proposal
    invSourcePdf = currentRluLightCount for every candidate proposal
    sum of active source probabilities equals 1.0 over the full current RLU
        dense domain
    invalid or zero-contribution lights remain in the proposal domain for the
        first proof and contribute zero targetPdf when sampled
    RTXDI helper sample/proposal count M is the number of streamed candidate
        proposals, not the RLU domain count unless the RTXDI helper call
        explicitly defines it that way

Pass:

    one light: source PDF is 1.0
    two uniform lights: each source PDF is 0.5
    N uniform lights: source PDF is 1/N
    RLU payload sourcePdf/sourceWeight is ignored by the first producer

Fail:

    analytic source PDF is read from manager sourcePdf when that value is zero
    Doom analytic payload sourcePdf=0 is used as the proposal PDF
    typed-range uniform, bounded per-range, payload-weighted, alias-table,
        portal, or ReGIR sampling is used as the first producer policy before
        the full-domain uniform RLU baseline is accepted
    source PDF changes after sampling without matching reservoir weight
    ReGIR, portal candidates, or old NEE output are used as the default source


Contract C-04B: Range-Stratified RLU Source PDF
-----------------------------------------------

Applies after PDFNEE-RLU-02 is accepted. This is the current policy-1 contract
for the checked-out replacement producer.

Required source:

    current RLU emissive typed range
    current RLU Doom analytic typed range

Required proof:

    raw current RLU typed ranges are used, not legacy sampling-gated ranges
    every non-empty sampled range states its rangeSampleCount
    totalProposalSamples is the sum of streamed range sample counts
    lightIndex is sampled uniformly inside each active range
    sourcePdf = rangeSampleCount / (rangeCount * totalProposalSamples)
    invSourcePdf = (rangeCount * totalProposalSamples) / rangeSampleCount
    selected reservoir identity remains dense current RLU lightIndex

Pass:

    one-class scenes match the stated rangeSampleCount/rangeCount PDF
    multi-range scenes produce a normalized sourcePdf table over all streamed
        proposals
    targetPdf, visibility, final contribution, RTXDI helper sequence, and
        stored current reservoir M are unchanged from PDFNEE-RLU-02

Fail:

    policy is documented as fixed 50/50 while the implementation uses
        rangeSampleCount/totalProposalSamples
    payload radiance, ReGIR cell weight, portal distance, or alias-table
        probability is used in this task
    sourcePdf is computed from rangeCount alone when multiple ranges are active
    selected light identity becomes class-local instead of dense RLU current
        lightIndex


Contract C-04C: Clean View-12 Analytic Parity
---------------------------------------------

Current blocker:

    PDFNEE/RLU current-frame Doom analytic lighting must match clean RTXDI DI
    view 12 before temporal consume or accumulator behavior is debugged.

Required comparison:

    same current RLU dense lightIndex
    same RLU record sourceIndex
    same current DoomAnalyticLights payload when clean view 12 uses sourceIndex
    same RAB_LightInfo fields: valid, position, radius, influenceRadius, area,
        radiance, weight, flags
    same RAB_SampleActiveRrxPolymorphicLight replay at the selected sample UV
    same solidAnglePdf
    same targetPdf
    same reflected radiance and final contribution

Pass:

    a Doom analytic selected through PDFNEE is neither absent nor visibly
        different from the same light in clean RTXDI DI view 12 with temporal
        disabled
    differences are explained by sourcePolicy sample choice/noise only, not by
        different light payload interpretation

Fail:

    legacy PDFNEE verifier view is used as the baseline instead of clean view 12
    active RRX unified-record loading rejects or reshapes a current Doom
        analytic that clean view 12 accepts through DoomAnalyticLights
    temporal, motion vectors, max history, depth thresholds, denoising, or
        fallback lighting are changed before current analytic parity passes


Contract C-05: Target PDF
-------------------------

Reference rbdoom file:

    neo/shaders/builtin/pathtracing/RtxdiBridge/RAB_LightTarget.hlsli

Required call:

    RAB_GetLightSampleTargetPdfForSurface(lightSample, surface)

Required shape:

    luminance(reflected BSDF radiance) / solidAnglePdf

Pass:

    targetPdf is positive for visible front-facing contributing samples
    zero targetPdf is loud and does not resolve as black albedo
    targetPdf uses the same lightSample replayed by final contribution

Fail:

    targetPdf is luminance(radiance * diffuse * ndotl) without division by
        solidAnglePdf
    targetPdf is a status or brightness ramp


Contract C-06: Final Direct-Light / NEE Contribution
----------------------------------------------------

Reference rbdoom file:

    neo/shaders/builtin/pathtracing/RtxdiBridge/PathTracer/RAB_PathTracerNee.hlsli

Required shape:

    reflectedRadiance *
    RTXDI_GetDIReservoirInvPdf(reservoir) /
    max(lightSample.solidAnglePdf, epsilon) *
    visibility

Pass:

    one-light output matches brute-force direct-light reference within expected
        sample noise
    overlap output remains stable in mean and does not strobe between light
        identities

Fail:

    resolve omits division by solidAnglePdf
    resolve uses albedo/beauty fallback
    resolve uses old rbdoom NEE contribution output
    resolve uses a temporal or luminance average to hide current-frame error


Contract C-07: Visibility
-------------------------

Reference rbdoom files:

    neo/shaders/builtin/pathtracing/RtxdiBridge/RAB_Visibility.hlsli
    pathtrace visibility helper used by the selected pass

Required proof:

    visibility policy is stated for each rung

Pass:

    one-light proof may use forced-visible only if explicitly marked
    real-light proof must have a visible mode and a traced-visibility mode
    visibility rejection is loud and separable from PDF failure

Fail:

    hidden fallback lighting fills rejected samples
    visibility failure is reported as PDF success


Contract C-08: Verifier Output Ownership
----------------------------------------

Required output:

    every pixel is written by the replacement producer/output path
    invalid state is explicit
    no fallback scene lighting underneath
    no temporal accumulation during current-frame proof

Pass:

    temporal-off noisy output is current-frame direct lighting or a clean
        current reservoir page
    black/invalid pixels are explainable by explicit invalid states

Fail:

    empty samples reveal base game lighting or albedo
    status colors are treated as estimator proof
    old PDFNEE/NEE output supplies the useful result


Contract C-09: Handoff To Clean RTXDI DI
----------------------------------------

Only after current-frame proof:

    feed verified current reservoir and replay contract into clean RTXDI DI
    view 12 consumer

Pass:

    temporal off preserves verified current-frame mean
    temporal on reduces noise without changing mean brightness

Fail:

    temporal integration begins before C-01 through C-08 pass
    selected light identity is not the dense RLU current-domain index consumed
        by the clean RLU path
    old external PDFNEE analytic-only identity mapping is treated as the
        replacement handoff
