PDF + NEE Verifier CVar Contract
================================

Purpose
-------

Reserve a small, standalone CVar surface for current-frame PDF + NEE
validation. These CVars must not control temporal accumulation.


Naming
------

Use this prefix unless a manager explicitly changes it:

    r_pathTracingRestirPdfNeeVerifier*


Required CVars
--------------

r_pathTracingRestirPdfNeeVerifierEnable

    0 = disabled
    1 = route standalone verifier output to primary path/test output

    Default: 0


r_pathTracingRestirPdfNeeVerifierView

    0 = disabled
    1 = route/status sentinel
    2 = raw flat diffuse direct-light output
    3 = selected light identity
    4 = PDF scalar view
    5 = final contribution scalar view
    6 = invalid-state classifier
    7 = one-light comparison helper
    8 = overlap comparison helper

    Default: 0


r_pathTracingRestirPdfNeeVerifierLightMode

    0 = no lights / negative path
    1 = one synthetic calibration light
    2 = two synthetic overlapping lights
    3 = N synthetic overlapping lights
    4 = one fixed real analytic light
    5 = two fixed real overlapping analytic lights
    6 = full real analytic domain
    7 = emissive domain
    8 = unified / ReSTIR light-manager domain
    9 = ReGIR current-frame source distribution

    Default: 0

    Mode 9 consumes the ReGIR candidate cache as a current-frame source
    distribution. It requires a same-frame ReGIR build prepass and remains
    current-frame only.


r_pathTracingRestirPdfNeeVerifierSamples

    Candidate proposals per pixel for the current-frame reservoir.

    First proof default: 1
    Valid range: 1 to 32


r_pathTracingRestirPdfNeeVerifierVisibility

    0 = forced visible, calibration only
    1 = traced visibility

    Default: 1


r_pathTracingRestirPdfNeeVerifierDomain

    0 = split RAB domain
    1 = unified RAB domain
    2 = ReSTIR light-manager RAB domain

    Default: 0

    PDFNEE-01 starts with split RAB domain because it is the narrowest route
    through the existing shared RAB load/sample helpers without depending on
    light-manager activation.


r_pathTracingRestirPdfNeeVerifierDump

    0 = disabled
    1 = one-shot dump of active verifier route, light counts, and first failing
        contract

    Default: 0


PDFNEE-10 Reserved CVars
------------------------

r_pathTracingRestirPdfNeeVerifierAnalyticPortalProposal

    0 = existing full analytic domain behavior
    1 = use the PDFNEE-10 portal-prioritized analytic candidate pool for
        lightMode 6

    Default: 0


r_pathTracingRestirPdfNeeVerifierAnalyticPortalSteps

    Portal traversal depth used to rank analytic proposal candidates.

    Default: 4
    Valid range: 0 to 8


r_pathTracingRestirPdfNeeVerifierAnalyticProposalMaxLights

    Maximum analytic candidate entries uploaded for the portal-prioritized
    pool.

    Default: 64
    Valid range: 1 to 1024


r_pathTracingRestirPdfNeeVerifierAnalyticDefensiveSamples

    Full-authored-domain analytic candidates streamed per pixel in addition to
    portal-prioritized candidates. These keep valid non-local lights reachable
    when Doom portal areas are too coarse.

    Default: 1
    Valid range: 0 to 8


r_pathTracingRestirPdfNeeVerifierAnalyticPortalDump

    0 = disabled
    1 = one-shot dump of full analytic count, portal-pool count, defensive
        sample count, portal-depth bins, selected global analytic indices,
        per-source sourcePdf / invSourcePdf, and any empty-list fallback reason

    Default: 0


PDFNEE-11 ReGIR Current-Frame Source
------------------------------------

PDFNEE ReGIR consume uses:

    r_pathTracingRestirPdfNeeVerifierLightMode 9
    r_pathTracingReGIREnable 1
    r_pathTracingReGIRLightDomain 0, 1, or 2

First proof preset:

    r_pathTracingDebugMode 0
    r_pathTracingReGIREnable 1
    r_pathTracingReGIRLightDomain 2
    r_pathTracingRestirPdfNeeVerifierEnable 1
    r_pathTracingRestirPdfNeeVerifierLightMode 9
    r_pathTracingRestirPdfNeeVerifierView 2
    r_pathTracingRestirPdfNeeVerifierSamples 1
    r_pathTracingRestirPdfNeeVerifierVisibility 1
    r_pathTracingRestirPdfNeeVerifierDump 1

Mode 9 must consume ReGIR only as a current-frame source distribution. It must
not enable PDFNEE temporal consume, clean RTXDI temporal consume, spatial reuse,
best-light sampling, RRX pages, mode 56, denoiser/confidence/gradient paths, or
fallback beauty/albedo output.


Forbidden CVar Behavior
-----------------------

These verifier CVars must not:

    enable temporal reuse
    enable spatial reuse
    enable best-light sampling
    enable denoiser/confidence/gradient paths
    route through r_pathTracingDebugMode 56
    reuse r_pathTracingRestirPTDiDebugView as the proof surface
    change the known-good clean synthetic accumulator


Required Runtime Preset
-----------------------

For estimator proof:

    r_pathTracingRestirPdfNeeVerifierEnable 1
    r_pathTracingRestirPdfNeeVerifierView 2
    r_pathTracingRestirPdfNeeVerifierSamples 1
    r_pathTracingRestirPdfNeeVerifierVisibility 1

Temporal, spatial, best lights, denoiser, and fallback lighting must be off
or irrelevant to the verifier route.
